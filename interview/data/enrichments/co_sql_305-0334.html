Reels watch-completion is the single most important Reels metric — it's the input feature to the algorithm and the output measure of creator quality. The query buckets each play into completed / engaged / sampled / bounced based on the watched-fraction.

2. The SQL — banded completion via CASE

WITH classified AS (
  SELECT *,
    CASE
      WHEN watched_seconds * 1.0 / NULLIF(total_seconds, 0) >= 0.95 THEN 'completed'
      WHEN watched_seconds * 1.0 / NULLIF(total_seconds, 0) >= 0.50 THEN 'engaged'
      WHEN watched_seconds * 1.0 / NULLIF(total_seconds, 0) >= 0.10 THEN 'sampled'
      ELSE 'bounced'
    END AS engagement_band
  FROM reel_plays
)
SELECT engagement_band,
  COUNT(*) AS plays,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER (), 1) AS pct
FROM classified
GROUP BY engagement_band
ORDER BY plays DESC;

3. Strategic explanations

Why 95% not 100% for "completed"

End-card detection has timing jitter. A reel labeled "complete" at 100% watched-seconds requires a perfect sync between client heartbeats and server end events. 95% is the operational threshold — close enough to count.

The intro-3-seconds metric

Reels' make-or-break moment is the first 3 seconds. A separate metric: watched_seconds >= 3 as "passed-the-hook". Production combines: bounce-pct (under 10%) is too late; the actionable creator metric is hook-passed-pct.

The duration-bias

Long reels (60s) have lower completion-rate than short reels (10s) — easier to sit through 10 seconds. Comparing creators with mixed durations is unfair. Normalize:

GROUP BY engagement_band, CASE WHEN total_seconds <= 15 THEN 'short' ELSE 'long' END
The audio vs no-audio split

Reels played with sound on have ~3x completion vs muted scrolling. Splitting by audio state changes which creators look strong: TikTok-style audio-driven creators win on sound-on metric.

The algorithm-feedback loop

Completion-rate isn't just a creator-quality signal — it's an algorithm-input feature. Reels with high completion rate get amplified to the next 100x audience; that audience also has high completion → exponential lift. The query result drives the ranker.

Per-reel vs aggregate

The query above aggregates across all reels. The PM-actionable variant: per-reel buckets. Add reel_id to GROUP BY and HAVING ≥ 1000 plays — that's the creator dashboard view.
