Per-product variant ranking (best color × size combo) drives merchandising decisions. The query uses RANK to surface each product's #1 variant.

2. The SQL — top-variant per product

WITH ranked AS (
  SELECT product_id, color, size, SUM(units_sold) AS total_units,
    RANK() OVER (PARTITION BY product_id ORDER BY SUM(units_sold) DESC) AS variant_rank
  FROM sales GROUP BY product_id, color, size
)
SELECT product_id, color, size, total_units FROM ranked
WHERE variant_rank = 1 ORDER BY total_units DESC;

3. Strategic explanations

RANK vs DENSE_RANK vs ROW_NUMBER

RANK keeps ties (multiple variants tied at #1 all show). DENSE_RANK collapses gaps. ROW_NUMBER picks one arbitrarily. RANK is right here — if two variants tie, both deserve recognition.

Velocity vs absolute

Black Medium might be #1 absolute but launched 6 months ago. Yellow Small is #1 in last 30 days. Production filters by date for velocity ranking.

Margin vs units

High-units variants might be the discounted clearance items. High-margin variants are the merchandiser's priority. Switch to SUM(units_sold * margin).

The supply constraint

"Best variant" can be inflated by under-stock of competing variants. If Yellow Small was out-of-stock, Black Medium captured all sales. Honest analysis controls for inventory availability.

Cross-sell signal

The top variant per product feeds "frequently bought with" recommendations.

Senior signal: RANK preserves ties, filter by recency for velocity, weight by margin not units, control for inventory availability, and feed cross-sell recommendations.
