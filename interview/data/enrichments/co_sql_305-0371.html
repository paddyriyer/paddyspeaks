Spotify's playlist team measures three actions: listen / save / share. The composite tells a story: high listens + low saves = transient enjoyment; high saves + low shares = personal favorite; high shares = social signal.

1. The source data — playlist_actions

2. The SQL — multi-action conditional aggregation + ratio

SELECT playlist_id,
  SUM(CASE WHEN action_type = 'listen' THEN 1 ELSE 0 END) AS listens,
  SUM(CASE WHEN action_type = 'save'   THEN 1 ELSE 0 END) AS saves,
  SUM(CASE WHEN action_type = 'share'  THEN 1 ELSE 0 END) AS shares,
  COUNT(DISTINCT user_id) AS unique_users,
  ROUND(100.0 * SUM(CASE WHEN action_type = 'save' THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN action_type = 'listen' THEN 1 ELSE 0 END), 0), 1) AS save_per_listen_pct
FROM playlist_actions
GROUP BY playlist_id
HAVING COUNT(*) >= 3
ORDER BY save_per_listen_pct DESC;

3. Strategic explanations

The save/listen ratio — engagement quality

Industry baseline: 5-10% of listens lead to a save for a curated playlist; below 3% means the playlist isn't resonating. Above 15% = strong personal-fit signal, candidate for similar-playlist recommendations.

Reach vs depth — uniqueness

1000 listens from 10 users (100/user) is different from 1000 from 1000 users (1/user). The first is super-fan binge; the second is broad reach. COUNT(DISTINCT user_id) exposes this.

The growth signal — share count

Shares are forward-looking: users who share are creating new acquisition. Playlists with shares-per-listen > 1% are the editorial team's "next breakout" candidates.

The seasonal listener-segment

"Workout" playlists peak in January (resolutions); "holiday" peaks in December. Trending alone misleads if you don't account for season.

Senior signal: read the listen/save/share triangle as engagement-quality + reach-depth + acquisition-signal, calibrate ratio thresholds (5-10% save baseline), and isolate seasonality before declaring a "trend".
