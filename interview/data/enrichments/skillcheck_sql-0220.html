The Senior Manager questions counted a manager’s direct reports. Span of control is the harder cousin: everyone beneath a manager, directly or indirectly, however deep the org runs. Computing it — along with each person’s level in the hierarchy — takes two recursive passes over the same self-referential table.

Jeff is the CEO (manager_id NULL). The org runs four levels deep: Jeff → Andy/Dave → Beth/Carl/Dana → the four individual contributors.

2. The recursive solution

WITH RECURSIVE org_levels AS (
  SELECT emp_id, emp_name, 1 AS level
  FROM employees
  WHERE manager_id IS NULL
  UNION ALL
  SELECT e.emp_id, e.emp_name, ol.level + 1
  FROM employees e
  JOIN org_levels ol ON e.manager_id = ol.emp_id
),
reports AS (
  SELECT manager_id AS mgr_id, emp_id AS sub_id
  FROM employees
  WHERE manager_id IS NOT NULL
  UNION ALL
  SELECT r.mgr_id, e.emp_id
  FROM reports r
  JOIN employees e ON e.manager_id = r.sub_id
),
span_counts AS (
  SELECT mgr_id, COUNT(*) AS span_of_control
  FROM reports
  GROUP BY mgr_id
)
SELECT ol.emp_name,
       ol.level,
       COALESCE(sc.span_of_control, 0) AS span_of_control
FROM org_levels ol
LEFT JOIN span_counts sc ON sc.mgr_id = ol.emp_id
ORDER BY span_of_control DESC, ol.emp_name ASC;

4. Strategic explanations

Two recursive CTEs under one WITH RECURSIVE

WITH RECURSIVE introduces the whole CTE list; the keyword appears once. A CTE is recursive only if it references itself — org_levels and reports do, span_counts does not. All three coexist under the single RECURSIVE.

Level: carry an accumulator down from the root

An employee’s level is not stored — it is the number of hops from the CEO. org_levels anchors Jeff at 1, and each recursive step emits ol.level + 1 as it moves to the next tier. Seed a value, increment it per hop: that accumulator pattern is how recursion measures depth.

Span: a transitive closure, not a direct count

reports builds the (manager, subordinate) closure — every manager paired with every person anywhere below them. The anchor takes direct reports; the recursive term reaches their reports, and onward. COUNT(*) per manager is then the full subtree size: Jeff’s span is 9, the whole company. Counting direct reports only — the Senior Manager problem — would have given Jeff 2.

LEFT JOIN, and why level and span are independent

The four ICs manage no one, so span_counts has no row for them; LEFT JOIN plus COALESCE reports span 0 instead of dropping them. The two metrics measure different things: Beth (level 3) and Dave (level 2) both have a span of 2 — sitting higher in the org does not mean managing more people. Span ranks by subtree size; level ranks by distance from the top.

Senior signal: “everyone under X” is a transitive closure — a recursive CTE whose anchor is direct reports and whose recursive term keeps descending — not a COUNT of direct reports. Depth is a separate recursion that carries an incrementing level accumulator down from the root.
