TikTok comment threads can go deep (5-10 levels). Threads with deep nesting indicate conversation — the highest engagement signal. Recursive CTE walks the chain.

2. The SQL — recursive thread depth

WITH RECURSIVE thread AS (
  SELECT comment_id, parent_id, video_id, likes, 1 AS depth
  FROM comments WHERE parent_id IS NULL
  UNION ALL
  SELECT c.comment_id, c.parent_id, c.video_id, c.likes, t.depth + 1
  FROM comments c JOIN thread t ON c.parent_id = t.comment_id
)
SELECT video_id, MAX(depth) AS max_thread_depth, COUNT(*) AS total_comments,
  SUM(likes) AS total_likes
FROM thread GROUP BY video_id ORDER BY max_thread_depth DESC;

3. Strategic explanations

Conversation as engagement

Single-level (10 separate top-level comments) = bunch of reactions. Deep thread (one comment with 9 replies) = real discussion. Algorithm weights thread-depth higher than comment count.

Cycle protection

Bad data: A reply to B, B reply to A. Recursive CTE infinite-loops. Add WHERE t.depth < 50 as a safety.

Most-engaged thread per video

Total likes on a thread = the conversation's quality. Branching:

WITH thread_likes AS (...) SELECT video_id, MAX(branch_likes) AS top_thread_quality FROM ...
The reply-chain controversy signal

Deep threads on controversial topics get heavy moderation. TikTok's safety system monitors deep-nested comments for harassment patterns.

The performance reality

Recursive CTE is slow at scale (billion-comment table). Production materializes thread_summary nightly via Spark.

Senior signal: thread-depth as conversation signal, depth-cap for cycle protection, separate branch-likes for thread-quality, watch deep-threads for harassment, and acknowledge nightly materialization for scale.
