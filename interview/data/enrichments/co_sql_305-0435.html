Checkout funnel: cart → contact → shipping → payment → confirm. Each step loses customers; the worst step is the merchant's #1 fix.

2. The SQL — step-to-step + percent-of-top

WITH counts AS (
  SELECT step, COUNT(DISTINCT session_id) AS sessions
  FROM checkout_events GROUP BY step
)
SELECT step, sessions,
  LAG(sessions) OVER (ORDER BY step) AS prev_step,
  ROUND(100.0 * sessions * 1.0 / NULLIF(LAG(sessions) OVER (ORDER BY step), 0), 1) AS step_to_step_pct,
  ROUND(100.0 * sessions * 1.0 / FIRST_VALUE(sessions) OVER (ORDER BY step), 1) AS pct_of_top
FROM counts ORDER BY step;

3. Strategic explanations

FIRST_VALUE for the cleanest denominator

Top of funnel size = step 1. FIRST_VALUE grabs it and replicates on every row. Cleaner than self-join.

Two metrics — different stories

Step-to-step exposes the leakiest single transition. Pct-of-top shows the cumulative loss. Together: which gate to fix AND how much it'd recover.
