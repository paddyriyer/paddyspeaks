APNs (Apple Push Notification service) processes hundreds of billions of notifications per day. The 5-second SLA is the published target. The query computes per-app delivery latency distribution + within-5s rate.

1. The source data — push_events

notification_idqueued_atdelivered_atapp_id
N_00110:00:0010:00:02APP_01

2. The SQL — latency CTE + within-SLA rate

WITH latencies AS (
  SELECT app_id,
    CAST((julianday(delivered_at) - julianday(queued_at)) * 86400 AS INTEGER) AS latency_seconds
  FROM push_events
  WHERE delivered_at IS NOT NULL
)
SELECT app_id,
  COUNT(*) AS deliveries,
  ROUND(AVG(latency_seconds), 1) AS avg_latency_sec,
  MAX(latency_seconds) AS max_latency_sec,
  SUM(CASE WHEN latency_seconds <= 5 THEN 1 ELSE 0 END) AS within_5s_count,
  ROUND(100.0 * SUM(CASE WHEN latency_seconds <= 5 THEN 1 ELSE 0 END) / NULLIF(COUNT(*), 0), 1) AS within_5s_pct
FROM latencies
GROUP BY app_id
ORDER BY within_5s_pct DESC;

3. Strategic explanations

The seconds conversion — *86400

julianday-diff is in days. Multiply by 86400 for seconds. Memorize: 1 / 60 / 1440 / 86400 for days/hours/minutes/seconds.

SLA reporting — the within-N count

"Average latency 1.2s" can hide that 5% of pushes take 30s. The within-5s percentage is the SLA. Apple's published target is 99% within 5 seconds. Below 99% triggers SRE investigation.

The undelivered-notification handling

Pushes can fail (token invalid, device offline). The query filters with WHERE delivered_at IS NOT NULL — measuring only delivered ones. To measure deliverability, compute delivery-rate separately:

COUNT(delivered_at) * 1.0 / COUNT(*) AS deliverability
Per-region latency divergence

APNs has regional gateways. Latencies vary by region: US sub-1s, India 2-3s typical. Slice:

GROUP BY app_id, region
Time-of-day spike

Latency spikes at peak (e.g., when a major news event triggers mass push from many apps simultaneously). Production monitors per-hour to catch capacity-planning gaps.

The priority-class twist

APNs has high-priority (immediate) and low-priority (battery-aware, delayed) classes. Low-priority pushes legitimately take minutes — they shouldn't be counted against the 5s SLA.

Senior signal: compute within-SLA rate not just average, separate latency from deliverability, slice by region + time-of-day, and exclude low-priority class from SLA evaluation. Production target 99% within 5s is the published bar.
