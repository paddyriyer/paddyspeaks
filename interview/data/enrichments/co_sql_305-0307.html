The single most-watched chart at Netflix in any given week is trial-to-paid conversion. The free trial is the funnel; whatever fraction of new signups flips to paid is the top-of-revenue input. Cohort it by signup week and you get the leading indicator for next-quarter revenue. The query is conditional aggregation; the senior conversation is what counts as "converted" and how to handle the in-flight cohorts.

2. The SQL — conditional aggregation cohorted by week

SELECT
  DATE_TRUNC('week', signup_date) AS cohort_week,
  COUNT(*) AS signups,
  COUNT(first_paid_date) AS converted_to_paid,
  COUNT(*) FILTER (WHERE first_paid_date IS NULL AND cancel_date <= trial_end_date) AS dropped_in_trial,
  ROUND(100.0 * COUNT(first_paid_date) / NULLIF(COUNT(*), 0), 1) AS trial_to_paid_pct
FROM subscriptions
GROUP BY DATE_TRUNC('week', signup_date)
ORDER BY cohort_week;

3. Strategic explanations

Why COUNT(first_paid_date) works as "converted"

COUNT(col) counts non-NULL values; rows where the user never paid have first_paid_date IS NULL and don't contribute. This is the cleanest "count rows matching a NULL-vs-NOT-NULL predicate" idiom — shorter than SUM(CASE WHEN first_paid_date IS NOT NULL THEN 1 END). Equivalent forms:

COUNT(*) FILTER (WHERE first_paid_date IS NOT NULL)            -- explicit, more readable
SUM(CASE WHEN first_paid_date IS NOT NULL THEN 1 ELSE 0 END)    -- portable, MySQL-friendly
COUNT(first_paid_date)                                          -- terse, only works for NULL/NOT-NULL
The in-flight-cohort trap — do NOT report this week's number

If today is May 1, the cohort that signed up the week of April 28 still has 27 days of trial left — their "trial_to_paid_pct" is artificially zero because nobody has paid yet. Report this week and you'll cause a panic. Production fix: only show cohorts where every member has had a chance to convert:

WHERE signup_date <= CURRENT_DATE - INTERVAL '30 days'   -- 30-day trial: wait for full window

Add a separate "in-flight" panel for current cohorts with the disclaimer "still converting".

Net vs gross conversion

The query reports gross conversion: anyone who ever flipped to paid. The CFO's actual question is net conversion at, say, day-90 — people who paid AND didn't cancel within 90 days. Different number, sometimes 30% lower:

COUNT(*) FILTER (
  WHERE first_paid_date IS NOT NULL
    AND (cancel_date IS NULL OR cancel_date > first_paid_date + INTERVAL '90 days')
) AS net_paid_at_d90
The cohort-week aggregation choice

Why week, not day or month?

  • Day: too noisy — weekend signups have different demographics. Lots of jitter.
  • Week: cancels out day-of-week effects, still granular enough to see campaigns.
  • Month: clean for finance, slow to detect changes. Lags real product changes by 4-6 weeks.

Production growth dashboards almost universally use weekly. DATE_TRUNC('week', date) defaults to Monday-start in Postgres; some dialects start on Sunday. Pin the week-start explicitly if cross-dialect parity matters.

Three drop-off buckets — not just "didn't convert"

"Did not convert" is too coarse. Three distinct failure modes:

Each bucket gets a different remediation: trial-cancel = onboarding flow improvement; auto-fall-off = payment-method confirmation page; convert-and-cancel = retention features. The dashboard should split these.

The cohort-curve follow-up — D7 / D14 / D30 conversion

Mature growth orgs don't stop at "ever converted" — they look at conversion velocity:

COUNT(*) FILTER (WHERE first_paid_date <= signup_date + INTERVAL '7 days')  AS d7_paid,
COUNT(*) FILTER (WHERE first_paid_date <= signup_date + INTERVAL '14 days') AS d14_paid,
COUNT(*) FILTER (WHERE first_paid_date <= signup_date + INTERVAL '30 days') AS d30_paid

Cohorts where d7-paid is unusually high are the "intent-buyer" segment — they signed up to pay, the trial was a formality. Cohorts where the curve is flat until d28 are the "trial-tourists" — only converting at trial-end-forced-decision.

The price-tier split

Netflix has multiple paid tiers (basic-with-ads, standard, premium). Conversion rate to any paid is one number; conversion rate to premium is another. Add a tier_at_conversion column and pivot.

Source attribution — the marketing follow-up

The marketing org wants conversion by acquisition source (organic, paid-search, paid-social, partner, referral) so they can re-allocate spend. Add a signup_source column to the GROUP BY:

GROUP BY DATE_TRUNC('week', signup_date), signup_source

Then the dashboard shows a side-by-side: which source drives quantity, which drives quality (high conversion %).

The denominator everybody forgets — trial completers

"Trial-to-paid" can mean two things:

  • "Of all signups, what % paid?" (the original query) — affected by trial-cancellers
  • "Of trial-completers, what % paid?" (more honest reflection of price acceptance)
COUNT(first_paid_date) * 100.0 /
NULLIF(COUNT(*) FILTER (WHERE cancel_date IS NULL OR cancel_date > trial_end_date), 0) AS trial_completer_to_paid_pct
Senior signal: use COUNT(col) as the terse "non-null" idiom but show the FILTER (WHERE) form for clarity, raise the in-flight-cohort artifact unprompted ("hide cohorts < 30 days old"), and split "didn't convert" into three meaningful buckets. Net-vs-gross conversion at D90 and the conversion-velocity curve are the over-the-bar finish.
