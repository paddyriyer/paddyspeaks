AMA (Ask Me Anything) sessions live or die by host responsiveness. Within-1-hour reply rate separates serious AMAs from absentee ones.

2. The SQL — within-1-hour reply rate

SELECT ama_id,
  COUNT(*) AS questions, COUNT(answered_at) AS answered,
  SUM(CASE WHEN julianday(answered_at) - julianday(asked_at) <= 1.0/24 THEN 1 ELSE 0 END) AS answered_within_1h,
  ROUND(100.0 * SUM(CASE WHEN julianday(answered_at) - julianday(asked_at) <= 1.0/24 THEN 1 ELSE 0 END)
              / NULLIF(COUNT(*), 0), 1) AS within_1h_pct
FROM ama_questions GROUP BY ama_id ORDER BY within_1h_pct DESC;

3. Strategic explanations

The 1.0/24 idiom for hour-window

1 hour in julianday-day units is 1/24. Memorize: 1/24 = hour, 1/1440 = minute, 1/288 = 5 min.

The "absentee AMA" antipattern

Host posts opening, gets 500 questions, answers 3, leaves. Reddit users hate this — Karma collapses, future AMAs lose trust. The metric flags it.

Question selection bias

Hosts answer easy / friendly questions, skip hard ones. answered / total hides this. Production splits by question controversy (downvote ratio).

Live vs delayed

Some AMAs are scheduled for "1 hour live, then trickle answers". Others are async. Comparing them is unfair; mark AMAs by format.

The answer-quality signal

Fast + short answers vs slow + thoughtful is the host-style choice. Production overlays answer-length to reveal style.

Senior signal: 1.0/24 for hour windows, identify absentee-AMA antipattern, watch for question-selection bias, distinguish live vs async format, and overlay answer-length for style.
