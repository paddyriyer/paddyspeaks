Airbnb's Superhost program is a quality signal — Superhosts get higher search ranking, +20% bookings, and a badge. The query checks 4 criteria simultaneously via a single CASE expression.

1. The source data — host_metrics

H_01154.9950

2. The SQL — multi-criteria CASE

SELECT host_id, completed_trips, avg_rating, response_rate_pct, cancellation_count,
  CASE
    WHEN completed_trips     >= 10
     AND avg_rating          >= 4.8
     AND response_rate_pct   >= 90
     AND cancellation_count  =  0
    THEN 'superhost' ELSE 'not_eligible'
  END AS status
FROM host_metrics
ORDER BY status, avg_rating DESC;

3. Strategic explanations

Multi-criteria AND — all four must hold

The four criteria are AND-joined: ALL must be true. A 4.9-rated host with 1 cancellation fails. This strictness is intentional — Superhost is a guarantee.

The "near-miss" segment

Hosts who fail ONE criterion are the highest-leverage marketing target — a small nudge gets them eligible. Surface them:

SELECT host_id,
  CASE WHEN completed_trips    < 10 THEN 'needs_more_trips' END,
  CASE WHEN avg_rating         < 4.8 THEN 'needs_better_reviews' END,
  CASE WHEN response_rate_pct  < 90 THEN 'needs_faster_response' END,
  CASE WHEN cancellation_count > 0 THEN 'cancellation_blocker' END
FROM host_metrics
The retroactive removal

Superhost is recalculated quarterly. A Superhost who cancels mid-quarter loses status. The query represents the latest snapshot; production tracks transitions over time.

The rating-trap: response volume

5.0-star with 3 reviews is statistically meaningless. Production adds a Bayesian smoothing — true rating moves toward 4.5 baseline as review count drops. Senior implementation:

(avg_rating * review_count + 4.5 * 5) / (review_count + 5) AS smoothed_rating
The criteria-tuning loop

Airbnb tweaks the 4 thresholds based on Superhost density. Too many → tighten (raise rating bar). Too few → loosen (drop trips bar to 5). The query result feeds the calibration.

Senior signal: combine multi-criteria via AND for the strict gate, surface the "near-miss" segment as the highest-leverage marketing target, apply Bayesian rating smoothing for low-review hosts, and connect to the threshold-tuning calibration loop. Quarterly retro removal is the operational nuance.
