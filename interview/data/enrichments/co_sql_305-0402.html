Pinterest's promoted pins (ads) come in formats: image / video / collection / story. Per-format CTR is the bidding-team's input — different formats command different CPC.

2. The SQL — per-format CTR with sample-size gate

SELECT pin_format,
  COUNT(*) AS impressions,
  SUM(clicked) AS clicks,
  ROUND(100.0 * SUM(clicked) / NULLIF(COUNT(*), 0), 2) AS ctr_pct
FROM promoted_pins GROUP BY pin_format
HAVING COUNT(*) >= 3
ORDER BY ctr_pct DESC;

The CPM-to-CPC bridge

Advertisers pay CPM (cost per mille); platform cares about CPC (cost per click) = CPM / (CTR × 10). Higher CTR → lower effective CPC for advertiser → more bid pressure → higher CPM. The query feeds the bidding model.

Saves-as-conversions

On Pinterest, "click" is one conversion; "save" is another. Save-through rate (saves / impressions) is often more valuable than CTR for retail advertisers. Production tracks both.

Format × audience interaction

Video performs better with younger audiences. Image does better with 35+. The CTR baseline must control for audience-segment.

Auto-bid vs manual-bid

Auto-bid uses Pinterest's algorithm to spend efficiently. Manual-bid relies on the marketer's CPM. Per-format CTR feeds the auto-bid model retraining.

Senior signal: name format-specific CTR baselines, surface saves as a Pinterest-specific conversion equal-weighted with clicks, control for audience-segment, and connect to auto-bid algorithm retraining.
