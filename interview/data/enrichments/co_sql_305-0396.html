Snap's AR filters are core differentiation. The query ranks filters by usage volume + reach (unique users) + share-of-total — the input to the editorial team's "what to feature" decisions.

2. The SQL — top-N + share aggregate

SELECT filter_id,
  COUNT(*) AS uses,
  COUNT(DISTINCT user_id) AS unique_users,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER (), 1) AS share_of_uses_pct
FROM filter_usage GROUP BY filter_id
ORDER BY uses DESC LIMIT 10;

3. Strategic explanations

Uses vs reach

10K uses from 1K users (10/user) = power-user filter. 10K from 8K (1.25/user) = broad-appeal filter. Both interesting; different editorial responses. Always emit both.

Editorial vs community filters

Snap-built filters (front-page) vs community Lens Studio filters (search). Different baselines. Slice:

JOIN filter_meta fm ON fm.filter_id = filter_usage.filter_id
GROUP BY filter_id, fm.source
Time-decay — the relevance window

"Top filters this week" is more useful than "all-time top". Production restricts to a recency window:

WHERE julianday('now') - julianday(used_at) <= 7
The seasonal spike

Halloween filters spike Oct 25-31. Christmas Dec 20-25. Valentines Feb 12-14. Without controlling for season, year-over-year usage comparisons mislead.

The brand-partnership tie-in

Brand-sponsored filters (Coca-Cola, Marvel) are revenue events. Their usage drives ad-rate negotiation. Production splits sponsored vs organic.

Senior signal: emit uses + unique_users for power-vs-broad signal, distinguish editorial vs community-source filters, apply recency windows for relevance, and surface seasonal + sponsorship dimensions.
