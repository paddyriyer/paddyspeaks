TikTok's algorithm amplifies sounds that are spreading. 5+ unique creators using a sound in 7 days = candidate for "trending" page. The query is a windowed creator-count.

2. The SQL — 7-day sound usage

WITH window_uses AS (
  SELECT sound_id, COUNT(*) AS uses_7d, COUNT(DISTINCT creator_id) AS unique_creators_7d
  FROM sound_uses
  WHERE julianday('2024-04-01') - julianday(used_at) BETWEEN 0 AND 7
  GROUP BY sound_id
)
SELECT sound_id, uses_7d, unique_creators_7d,
  CASE WHEN unique_creators_7d >= 5 THEN 'viral' ELSE 'normal' END AS classification
FROM window_uses ORDER BY unique_creators_7d DESC;

3. Strategic explanations

Unique creators >> total uses

1 creator using a sound 100 times is a personal favorite. 100 creators using it once each = trend. Diversity matters more than volume for "viral".

The bot-creator filter

Spam farms use a sound 1000x. Filter WHERE creator.is_legitimate; otherwise virality is gameable.

Velocity acceleration vs steady state

5 creators in last 7 days might be normal — same as last week. Real virality is acceleration:

uses_7d / NULLIF(uses_in_prior_7d, 0) AS growth_multiplier

3x+ growth is the "actually trending" signal.

The amplification feedback

"Viral" sounds get pushed to the Discover page → more creators see → more uses → more viral. Production caps the amplification rate to prevent runaway memes from drowning organic content.

The trademark risk

Some viral sounds are unauthorized music clips. Legal team monitors top-N viral sounds for licensing violations.

Senior signal: use unique-creators not raw uses, filter bot-creators upfront, distinguish steady-state from acceleration via growth_multiplier, and acknowledge legal review of trending audio.
