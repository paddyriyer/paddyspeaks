Same content posted to multiple subreddits = either cross-promotion (legit) or spam-farm (illegit). The query identifies posts appearing across 2+ subreddits.

2. The SQL — title-based cross-post detection

SELECT a.title,
  COUNT(DISTINCT a.subreddit_id) AS subreddits_count,
  GROUP_CONCAT(DISTINCT a.subreddit_id) AS subreddits
FROM posts a
GROUP BY a.title HAVING COUNT(DISTINCT a.subreddit_id) >= 2
ORDER BY subreddits_count DESC;

3. Strategic explanations

Title-match vs content-match

Titles match across subs is the simplest detection. URL match is stronger (content identity). Hash of body text catches reposted text. Production runs all three.

GROUP_CONCAT — SQLite's array-agg

SQLite's GROUP_CONCAT(DISTINCT col) = Postgres's STRING_AGG(DISTINCT col, ', ') = comma-separated list. Useful for showing the set inline.

The Reddit-built crosspost feature

Reddit has an explicit "crosspost" mechanism — link to original. The query flags independent cross-posting (same content, no link). Production excludes built-in crossposts.

The repost economy

Some users intentionally repost old content for karma. Time-since-original-post is the signal: posts re-appearing 6+ months later = repost farm.

Senior signal: escalate from title-match → URL-match → content-hash, use GROUP_CONCAT for inline subreddit list, distinguish legit cross-promo from karma farm, exclude Reddit's built-in crosspost mechanism, and watch repost-after-time as farm signal.
