Uber's Growth team segments riders by recency and frequency. "Riders with 3+ trips in the last 30 days" is the active rider cohort — they've crossed the habit threshold and are predictable revenue. The query is HAVING + a date-window filter.

1. The source data — rides

2. The SQL — date window + HAVING

SELECT rider_id, COUNT(*) AS rides_30d
FROM rides
WHERE julianday(started_at) >= julianday('2024-02-01') - 30
  AND julianday(started_at) <= julianday('2024-02-01')
GROUP BY rider_id
HAVING COUNT(*) >= 3
ORDER BY rides_30d DESC;

3. Strategic explanations

The "snapshot date" pattern

Hardcoding '2024-02-01' as the snapshot makes the result reproducible — same data, same answer regardless of when the query runs. Production analytics jobs always use a snapshot variable, not CURRENT_DATE. Set it via a CTE in real workflows:

WITH params(snapshot) AS (VALUES ('2024-02-01'))
SELECT ... WHERE julianday(started_at) BETWEEN
  julianday((SELECT snapshot FROM params)) - 30 AND julianday((SELECT snapshot FROM params))
The "3 rides" threshold — why this number

Empirically derived. Uber's published frequency curve has a clear knee at 3-4 trips/month: riders below that often churn within 90 days; riders above stick around 12+ months. The threshold should be calibrated per-market — Bangalore commute riders cross the habit threshold at 8+, NYC airport riders at 1.

The recency vs frequency split

"3 rides in 30 days" combines two signals. Better: split into R (days since last) and F (count of trips), the classic RFM framework. Different combinations need different campaigns:

The "type of rides" qualifier

3 airport runs in 30 days = high-LTV business traveler. 3 short rides in 30 days = different segment. Always split rides by trip-type (airport, work-commute, social, late-night) before forming cohorts.

The cohort-decay follow-up

Rolling the 30-day filter forward week-by-week shows cohort retention curves. Repeat the query for snapshot dates 2024-01-08, 01-15, 01-22, 01-29... and track the % of users staying in the active cohort. That curve is Uber's monthly board-deck slide.

Senior signal: use a snapshot date for reproducibility, calibrate the threshold from data not gut, and reframe the 30-day filter as recency × frequency RFM. The trip-type split and rolling cohort curve are the over-the-bar move.
