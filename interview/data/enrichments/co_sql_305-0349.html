Amazon runs hundreds of recommendation A/B tests concurrently. The query splits events by treatment bucket and computes CTR + add-to-cart rate, with a derived "add-to-cart-per-click" metric for funnel-stage diagnosis.

1. The source data — ab_events

user_idbucketrecommended_asinclickedadded_to_cart
U_001treatmentA_0111

2. The SQL — three rates per bucket

SELECT bucket,
  COUNT(*) AS impressions,
  ROUND(100.0 * SUM(clicked) / NULLIF(COUNT(*), 0), 2) AS ctr_pct,
  ROUND(100.0 * SUM(added_to_cart) / NULLIF(COUNT(*), 0), 2) AS atc_pct,
  ROUND(100.0 * SUM(added_to_cart) / NULLIF(SUM(clicked), 0), 2) AS atc_per_click_pct
FROM ab_events
GROUP BY bucket
ORDER BY bucket;

3. Strategic explanations

Three metrics — three different stories
metricasks
CTR"do users notice the rec?"
ATC"do recs lead to intent?"
ATC-per-click"are clicks high-intent?"

A treatment that boosts CTR but tanks ATC-per-click is clickbait — net negative. Always compute the funnel-stage ratio.

The lift calculation

"Treatment 5% CTR, control 4% CTR" — what's the lift? +25% relative, +1pp absolute. Both matter; one number lies. Senior dashboards report:

(treatment.ctr - control.ctr) AS abs_lift_pp,
(treatment.ctr - control.ctr) / NULLIF(control.ctr, 0) AS rel_lift
Statistical significance — the blocking gate

"Treatment looks 8% better" is meaningless without a CI. Use 2-proportion z-test:

1.96 * SQRT((p_t * (1-p_t) / n_t) + (p_c * (1-p_c) / n_c)) AS ci_pp_at_95

Promote the treatment only when |lift| > CI.

Sample-ratio mismatch (SRM) — the validity check

Treatment should have ~50% of impressions vs control. If the actual ratio is 47/53, randomization broke. SRM check:

SUM(CASE WHEN bucket = 'treatment' THEN 1 ELSE 0 END) * 1.0 / COUNT(*) AS treatment_share

Off by > 1pp = invalid test, void the result, debug the bucketing logic.

The novelty effect

New recs initially boost CTR (curiosity) then plateau. A 7-day test shows fake lift; a 28-day test shows truth. Production runs tests for full novelty + recency cycles before declaring a winner.

Heterogeneous treatment effects

Average lift hides segment-specific effects. Treatment may help new users (+5%) but hurt power users (-3%). Slice by user-tenure for honest assessment.

Senior signal: compute CTR / ATC / ATC-per-click as a funnel chain, emit absolute + relative lift + CI, run SRM as a blocking gate, and pre-empt the novelty + heterogeneous-effect traps.
