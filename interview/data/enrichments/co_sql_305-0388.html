LinkedIn Jobs' funnel: saved → applied → recruiter-viewed. The query measures conversion at each stage, exposing where candidates drop.

2. The SQL — sequential conditional aggregation

WITH per_pair AS (
  SELECT member_id, job_id,
    SUM(CASE WHEN action_type = 'saved'   THEN 1 ELSE 0 END) AS saved,
    SUM(CASE WHEN action_type = 'applied' THEN 1 ELSE 0 END) AS applied,
    SUM(CASE WHEN action_type = 'recruiter_viewed' THEN 1 ELSE 0 END) AS recruiter_viewed
  FROM job_actions GROUP BY member_id, job_id
)
SELECT
  COUNT(*) AS member_job_pairs,
  SUM(CASE WHEN saved >= 1 THEN 1 ELSE 0 END) AS saved_step,
  SUM(CASE WHEN applied >= 1 THEN 1 ELSE 0 END) AS applied_step,
  SUM(CASE WHEN recruiter_viewed >= 1 THEN 1 ELSE 0 END) AS recruiter_viewed_step,
  ROUND(100.0 * SUM(CASE WHEN applied >= 1 THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN saved >= 1 THEN 1 ELSE 0 END), 0), 1) AS save_to_apply_pct
FROM per_pair;

3. Strategic explanations

The (member, job) pair as the unit

The funnel is per-job-per-member. GROUP BY member_id, job_id creates one row per pair. Action presence (≥ 1) fires per-step.

Save → Apply rate baseline

Industry: 40-50% of saved jobs lead to apply within 14 days. Lower than 30% = save-feature is being misused (bookmarking) or the apply-flow is broken.

Apply → Viewed-by-recruiter rate

~25-35% of apps get recruiter views. Variance comes from job posting quality, recruiter velocity, candidate quality. Below 15% = recruiter-side bottleneck (low Recruiter seat usage).

Easy Apply vs Standard Apply

Easy Apply (1-click) has 10x apply-rate but 5x lower recruiter-view rate (lower-quality candidates). Production splits by application type.

Time-to-apply vs time-to-view

Median time-to-apply: 2 days post-save. Median time-to-recruiter-view: 5 days post-apply. Long lag → candidates lose interest. Production tracks median latencies.

Senior signal: use (member, job) pair as the funnel unit, cite 40-50% save→apply baseline, separate Easy Apply from Standard, and surface latency between stages as the disengagement risk.
