Spotify's adaptive bitrate (ABR) system shifts audio quality based on network conditions. The query uses LAG to detect bitrate transitions per session — high churn signals network instability.

2. The SQL — LAG transition detection

WITH transitions AS (
  SELECT session_id, event_at, bitrate_kbps,
    LAG(bitrate_kbps) OVER (PARTITION BY session_id ORDER BY event_at) AS prev_bitrate
  FROM playback_events
)
SELECT session_id,
  COUNT(*) AS events,
  SUM(CASE WHEN prev_bitrate IS NOT NULL AND bitrate_kbps != prev_bitrate THEN 1 ELSE 0 END) AS quality_changes,
  MIN(bitrate_kbps) AS lowest_bitrate,
  MAX(bitrate_kbps) AS highest_bitrate
FROM transitions
GROUP BY session_id
ORDER BY quality_changes DESC;

Each transition between tiers is an ABR event. Many transitions per session = unstable network → poor user experience.

Drops vs upgrades — different signals

Drops (320 → 160 → 96) signal network degradation; upgrades (96 → 160 → 320) signal recovery. Splitting reveals SLA breaches:

SUM(CASE WHEN bitrate_kbps < prev_bitrate THEN 1 END) AS drops,
SUM(CASE WHEN bitrate_kbps > prev_bitrate THEN 1 END) AS upgrades
The first-event NULL handling

The first event in each session has prev_bitrate IS NULL — it's not a transition. The CASE filters with prev_bitrate IS NOT NULL.

Per-network-type cut

Cellular sessions have 5x more transitions than WiFi. Production splits by network_type for fair benchmarking.

The lossless-tier upsell

Premium-only "Very High" 320kbps and HiFi (lossless, 1411kbps in pilot regions) drive upgrades. Sessions consistently at 320 in HiFi-eligible markets are upsell targets.
