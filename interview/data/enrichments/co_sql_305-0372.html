Discover Weekly is Spotify's flagship algorithm. The single quality metric: discovery rate = % of newly-played tracks that the user later saved to library. The query is a time-bounded LEFT JOIN attribution.

2. The SQL — 7-day attribution window

SELECT
  COUNT(DISTINCT d.user_id || '|' || d.song_id) AS discovered_pairs,
  COUNT(DISTINCT CASE WHEN s.song_id IS NOT NULL THEN d.user_id || '|' || d.song_id END) AS saved_pairs,
  ROUND(100.0 * COUNT(DISTINCT CASE WHEN s.song_id IS NOT NULL THEN d.user_id || '|' || d.song_id END)
              / NULLIF(COUNT(DISTINCT d.user_id || '|' || d.song_id), 0), 1) AS discovery_save_rate_pct
FROM discover_plays d
LEFT JOIN library_saves s
  ON s.user_id = d.user_id
 AND s.song_id = d.song_id
 AND julianday(s.saved_at) >= julianday(d.played_at)
 AND julianday(s.saved_at) - julianday(d.played_at) <= 7;

3. Strategic explanations

The 7-day window

Discover Weekly refreshes Mondays; the 7-day window captures saves that happen within the same playlist cycle. Longer windows over-attribute; shorter under-attribute the slow-burn discoveries.

The string-concat dedup key

user_id || '|' || song_id creates a unique pair-key for COUNT(DISTINCT). Cleaner than COUNT(DISTINCT user_id, song_id) which only works in some dialects.

The published target

Spotify reports ~30% Discover-save rate publicly. Users who save 5+ tracks per week from Discover have 90%+ retention; the metric drives the recommender's loss function.

Save vs play-again — different conviction

"Saved to library" is high conviction. "Played again within 7 days" is medium. "Listened past 30 seconds" is low. Production tracks all three; this query measures the strongest signal.

The genre-affinity feedback

If a user's discovery-save-rate is 35% in Indie but 10% in EDM, the algorithm should weight Indie heavier in next week's playlist. The query result feeds the personalization model.

Senior signal: 7-day window matches the playlist refresh cycle, use string-concat for portable composite-key dedup, contrast save (high conviction) vs play-again (medium) vs 30s-listen (low), and connect to the personalization feedback loop.
