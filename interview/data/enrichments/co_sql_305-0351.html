Amazon's Sponsored Products business is a $30B+ revenue line. Attribution decides what % of each purchase to credit to ads. The query handles two windows: click-through (7-day) and view-through (24-hour).

2. The SQL — windowed attribution split

SELECT
  SUM(CASE WHEN i.clicked = 1 AND julianday(p.purchased_at) - julianday(i.shown_at) BETWEEN 0 AND 7 THEN 1 ELSE 0 END) AS click_through_attr,
  SUM(CASE WHEN i.clicked = 0 AND julianday(p.purchased_at) - julianday(i.shown_at) BETWEEN 0 AND 1 THEN 1 ELSE 0 END) AS view_through_attr,
  COUNT(*) AS attributed_pairs
FROM ad_impressions i
JOIN purchases p ON p.user_id = i.user_id AND p.asin = i.asin
  AND julianday(p.purchased_at) > julianday(i.shown_at)
  AND julianday(p.purchased_at) - julianday(i.shown_at) <= 7;

3. Strategic explanations

Click-through vs view-through — different conviction
typewindowquality
Click-through7 daysstrong — user actively engaged
View-through1 dayweak — user just SAW the ad

Different windows reflect different conviction levels. Combining them inflates attribution; reporting separately is honest.

The double-count danger

If a user sees + clicks + buys within 24 hours, both branches fire. The CASE conditions (clicked=1 vs clicked=0) make them mutually exclusive — same row contributes to one bucket only.

Last-touch vs multi-touch

The query above is last-touch: each purchase credits the latest ad impression in window. Production also runs multi-touch (split credit across all touches in the path) — different attribution model, very different revenue numbers.

Incrementality — the senior reframe

Attribution measures revenue ASSOCIATED with ads. Incrementality measures revenue that wouldn't have happened without ads. They diverge: a power user who buys daily would've bought anyway. True incremental ROAS is often 30-50% of attributed ROAS. Mention this — it's the senior reframe.

The asin-match strictness

The query requires exact ASIN match. But users often see ad for ASIN-A, click, then buy ASIN-B (variant or substitute). Attribution can be widened to category-match for "halo" effects. Production splits attribution into "exact-asin" and "halo" buckets.

Senior signal: distinguish click-through (7d) vs view-through (1d) windows, prevent double-count via mutually-exclusive CASE branches, contrast last-touch vs multi-touch, and reframe attribution vs incrementality. The exact-vs-halo ASIN match is the over-the-bar nuance.
