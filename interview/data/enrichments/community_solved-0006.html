A Community follow-up to the year-over-year delta question: instead of listing individual increase rows, return the brands that grew in every year — the all-time risers, with no down year. The reference solution flips the logic to NOT EXISTS ("keep the brand if no year shows a drop"), which is the elegant way to express "for all" in SQL. The interviewer wants to see whether you know why the negation is cleaner than a positive check.

1. The source data — brand_amount

2021apple150
2022apple180
2020nike80
2021nike60

2021sony200

apple climbs every year; nike dips in 2021 (80→60) before recovering — one bad year disqualifies it; sony has a single year, a one-row group with no comparison possible at all.

2. The SQL — NOT EXISTS with a "down year" probe

WITH t1 AS (
    SELECT brand, year AS current_year, (year - 1) AS previous_year, amount,
           LAG(amount, 1, 0) OVER (PARTITION BY brand ORDER BY year ASC) AS amount_previous_yr,
           (amount - LAG(amount, 1, 0) OVER (PARTITION BY brand ORDER BY year ASC)) AS amount_change
    FROM brand_amount
)
SELECT * FROM brand_amount
WHERE NOT EXISTS (
    SELECT brand FROM t1
    WHERE amount_change < 0 AND brand_amount.brand = t1.brand
);

t1 computes each brand-year's delta with LAG. The outer query keeps a row only if its brand has no year in t1 with a negative delta — "no down year anywhere" implies "up every year".

3. The resulting analysis

apple survives — 100, 150, 180, no drop — and all three of its rows are returned. nike is gone: its 2021 row has amount_change = 60 - 80 = -20 < 0, so the NOT EXISTS probe finds a match and excludes every nike row. sony also survives, but for a subtler reason worth unpacking below.

4. Strategic explanations

Why NOT EXISTS is the natural "for all" pattern

SQL has no direct "every row satisfies P" quantifier. The standard trick is double negation: "every year increased" ≡ "there does not exist a year that decreased". NOT EXISTS expresses that literally, and it short-circuits — the moment one down year is found, the brand is rejected. A positive formulation ("count of up years = count of comparison years") works too but needs careful off-by-one counting; the negation sidesteps that.

The single-row group — why sony slips through

sony has one year. There is no prior year, so it can have no down year, so NOT EXISTS finds nothing and keeps it. Is a brand with one data point "increasing every year"? Almost certainly not — it is "vacuously true", the classic boundary of universally-quantified logic. Decide deliberately: if a brand must have at least two years to qualify, add a guard:

AND (SELECT COUNT(*) FROM brand_amount b2
     WHERE b2.brand = brand_amount.brand) > 1

Naming the vacuous-truth case unprompted is the senior tell.

The flat-year subtlety — < 0 vs <= 0

This solution rejects a brand only on a strict decrease (amount_change < 0). A brand that goes 100 → 150 → 150 — flat in the last year — has no negative delta, so it is kept. But "increased every year" arguably rules out a flat year. If strict growth is required, the probe must be amount_change <= 0. This is precisely the difference between this solution and its sibling (which uses an amount > LAG flag and so excludes flat years). Confirm with the interviewer whether flat counts.

The LAG(..., 1, 0) default and the first year

The default 0 makes every brand's first-year delta equal to its (positive) first amount — never negative — so the debut year never triggers a false rejection. Here that is harmless. But it is harmless by luck: if a brand could legitimately start at a negative amount, the default would matter. Cleaner intent is LAG(amount, 1) with no default, letting the first year be NULL (and NULL < 0 is unknown, so it never matches the probe either).

Output shape: rows vs. brand names

SELECT * FROM brand_amount returns every row of qualifying brands. If the question wants just the brand list, use SELECT DISTINCT brand — or build the answer entirely from t1 with a GROUP BY brand HAVING MIN(amount_change) >= 0, which is arguably the tightest formulation of all.

Senior signal: recognise NOT EXISTS as the idiomatic encoding of "for all", flag that single-year brands pass vacuously and decide whether that is wanted, and call out that < 0 tolerates a flat year whereas the question may demand strict growth.
