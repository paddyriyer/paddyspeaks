Amazon poses "Container With Most Water" to test one specific instinct: recognizing when a brute-force O(n²) pair scan can be collapsed to a single O(n) two-pointer sweep by a greedy elimination argument. The real challenge is not coding the loop — it is proving that moving the shorter wall is always safe.

1. The input data

h = [1, 8, 6, 2, 5, 4, 8, 3, 7]

Nine vertical lines of these heights. Water held between lines i and j is (j - i) * min(h[i], h[j]) — width times the shorter wall.

2. The approach

Two pointers start at the extremes: l = 0, r = len(h) - 1 — the widest possible container. Each step computes the area for the current pair and updates best.

Then the greedy move: advance the pointer at the shorter wall inward. Why is this safe? The area is capped by the shorter wall. Keeping the shorter wall and shrinking the width can only ever decrease area — so every pair involving that shorter wall is already dominated by the area we just recorded. Discarding it loses nothing. Moving the taller wall instead would throw away potentially better pairs. The loop ends when the pointers meet.

4. Strategic explanations

Complexity

O(n) time — each iteration advances exactly one pointer, so the pointers together travel n steps. O(1) space. The brute force over all pairs is O(n²); the interviewer wants the linear collapse and the correctness argument.

The idiomatic alternative — there isn't a cleaner one, but tighten the tie case

The two-pointer sweep is already optimal. The one nuance: when h[l] == h[r], the code moves r (the else branch). Moving either is fine for correctness — both walls are equally limiting — but you could move both at once as a micro-optimization, since no pair using either as the shorter wall can beat the current area. Mention it; do not over-engineer it.

The edge case — fewer than two lines

[] or [5]: l < r is false immediately, so best stays 0 — correct, you cannot form a container. Heights of 0 are valid input and simply contribute 0 area. All-equal heights still work, narrowing width each step.

The follow-up — "Trapping Rain Water"

The classic escalation swaps the objective: instead of one big container, sum water trapped above every bar. Same two-pointer skeleton, but now you track left_max and right_max and accumulate. Interviewers love this pairing because it reuses the pointer idea with a harder invariant.

The trap — moving the taller wall

If you advance whichever pointer has the taller wall, you can step past the optimal pair and report a smaller area. The fix is not "try both" (that is O(n²) again) — it is the elimination proof: the shorter wall caps the area, so it is the one that can be safely retired. A candidate who moves the wrong pointer has memorized the loop without understanding it.

Senior signal: do not just write the two-pointer loop — prove the greedy step. Saying "the shorter wall bounds the area, so any other pair using it is already dominated" is the sentence that demonstrates real understanding rather than pattern recall.
