Reddit moderators have wide latitude to remove submissions. Removal rate per subreddit exposes the moderation style — light-touch vs over-moderated. Both extremes hurt growth.

2. The SQL — removal rate + tier classification

SELECT subreddit_id,
  COUNT(*) AS submissions, SUM(removed) AS removed,
  ROUND(100.0 * SUM(removed) / NULLIF(COUNT(*), 0), 1) AS removal_rate_pct,
  CASE
    WHEN 100.0 * SUM(removed) / NULLIF(COUNT(*), 0) >= 30 THEN 'over_moderated'
    WHEN 100.0 * SUM(removed) / NULLIF(COUNT(*), 0) >= 10 THEN 'active'
    ELSE 'light_touch'
  END AS mod_style
FROM submissions GROUP BY subreddit_id HAVING COUNT(*) >= 3
ORDER BY removal_rate_pct DESC;

3. Strategic explanations

The 30%/10% style cutoffs

Calibrated from Reddit's published mod stats. Above 30% = users hit the wall, churn out. Below 10% = spam takes over. 10-30% is the productive band.

Auto-mod vs human

Removals split: auto-mod (rule-based) + human-mod (judgment). Auto-mod removals are not controversial. Human removals are the contentious slice. Production splits.

Rule-violation distribution

Removal reasons: spam, rule violation, low-effort, off-topic. Each different policy. Production groups by reason.

The shadow-removal practice

"Removed but visible to author" is a Reddit pattern. Production differentiates from full removal.

Restoration rate

Some removals get restored on appeal. The restoration rate flags over-aggressive auto-mod.

Senior signal: 10-30% productive removal band, split auto-mod vs human, group by reason, handle shadow-remove distinction, and track restoration rate as auto-mod calibration signal.
