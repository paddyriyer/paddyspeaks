An HR system wants to know which departments have the deepest salary parity at the top — the most people who all share the maximum salary in their department. The interviewer is probing the double-ranking pattern: first RANK to find who is tied at the top within each department, then an outer aggregation to count those ties, then a final filter to find the department(s) with the largest count (with ties at the dept level kept).

Eng: max salary = 200, tied by emp_ids 1, 2, 3 (3 top earners). Sales: max = 300, tied by 5 and 6 (2 top earners). Ops: max = 120, only emp 8 (1 top earner). HR: max = 90, tied by 10, 11, 12 (3 top earners). Both Eng and HR tie for the highest top-earner count of 3.

2. The SQL — RANK then count then max-filter

WITH ranked AS (
  SELECT dept, salary,
         RANK() OVER (PARTITION BY dept ORDER BY salary DESC) AS rk
  FROM employee
),
per_dept AS (
  SELECT dept, COUNT(*) AS top_earner_count
  FROM ranked
  WHERE rk = 1
  GROUP BY dept
)
SELECT dept, top_earner_count
FROM per_dept
WHERE top_earner_count = (SELECT MAX(top_earner_count) FROM per_dept)
ORDER BY dept;

4. Strategic explanations

Why RANK and not DENSE_RANK or ROW_NUMBER

RANK assigns the same rank to all rows that share the maximum salary within the department. rk = 1 therefore captures every tied top earner. DENSE_RANK behaves identically for the max (rank 1 goes to all highest-salary rows regardless of function), so either works here. ROW_NUMBER would assign distinct numbers even to ties, leaving only one "top earner" per department — incorrect.

The subquery vs. window function for the outer max

The final filter uses a scalar subquery SELECT MAX(...) FROM per_dept. An equivalent window approach would add RANK() OVER (ORDER BY top_earner_count DESC) AS outer_rk to per_dept and filter where outer_rk = 1. Both are correct; the subquery is marginally simpler. Note that using LIMIT 1 instead of the subquery would silently drop the Eng–HR tie, which is the key edge case this question tests.

COUNT(*) vs COUNT(salary) in the second CTE

After filtering rk = 1, all retained rows have a non-NULL salary (salary is defined as INT). COUNT(*) and COUNT(salary) are equivalent here. In general, prefer COUNT(*) when counting rows rather than non-NULL values in a specific column — it is clearer in intent.

Departments with a single employee

If a department has one employee, that person is trivially the sole top earner with top_earner_count = 1. The query handles this correctly without any special case.

Dialect notes

The RANK window function and scalar subquery pattern are ANSI SQL and work in PostgreSQL, Snowflake, BigQuery, SQL Server, and MySQL 8+. BigQuery requires the outer query to be a standard SELECT — no special changes. SQL Server supports CTEs with window functions identically. In MySQL 5.7 (no window functions), you would compute the max salary per department with a self-join and count via a nested GROUP BY.

Senior signal: use RANK() OVER (PARTITION BY dept ORDER BY salary DESC) (not ROW_NUMBER) to capture all salary ties at rank 1, aggregate those into a per-dept count, and use a scalar MAX subquery (not LIMIT 1) to retrieve all departments that tie for the highest count.
