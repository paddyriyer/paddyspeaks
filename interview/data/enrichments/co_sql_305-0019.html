Trivial-looking query (DISTINCT on the returns table) hides the actual senior conversation: which definition of "customer with returns" does the marketing brief mean? Unique returners ever? 4. Strategic explanations

DISTINCT vs GROUP BY — which is "right"?

For a single column with no aggregation, both produce identical results. Style varies by team; semantically:

SELECT DISTINCT customer_id FROM returns;
-- equivalent
SELECT customer_id FROM returns GROUP BY customer_id;

Use GROUP BY the moment you want to add an aggregate — "how many returns each?":

SELECT customer_id, COUNT(*) AS return_count
FROM returns GROUP BY customer_id
ORDER BY return_count DESC;

For our data: C_01 = 3 returns. That's the serial-returner signal.

The "with at least N returns" threshold

Marketing's actual ask is rarely "anyone who ever returned" — it's "frequent returners". HAVING:

SELECT customer_id
FROM returns
GROUP BY customer_id
HAVING COUNT(*) >= 3;

Returns just C_01.

The return-rate denominator — the senior reframe

"Number of returns" is misleading without context. A customer with 50 orders and 5 returns has a 10% return rate; a customer with 2 orders and 1 return has a 50% return rate. The latter is the actually-problematic profile. Join to orders:

WITH agg AS (
  SELECT o.customer_id,
    COUNT(*) AS orders,
    COUNT(DISTINCT r.return_id) AS returns
  FROM orders o
  LEFT JOIN returns r ON r.order_id = o.order_id
  GROUP BY o.customer_id
)
SELECT customer_id, orders, returns,
  ROUND(100.0 * returns / NULLIF(orders, 0), 1) AS return_rate_pct
FROM agg
WHERE orders >= 5             -- only customers with enough volume to be statistically meaningful
ORDER BY return_rate_pct DESC;

The WHERE orders >= 5 filter is the senior touch — without it, the top of the leaderboard is dominated by customers with one order and one return at 100%.

The 90-day window — recency matters

Lifetime return behaviour is usually less actionable than recent. Recent returners get the "we're so sorry, here's a credit" email; 5-year-old returners get nothing.

SELECT DISTINCT customer_id
FROM returns
WHERE return_date >= CURRENT_DATE - INTERVAL '90 days';
The "left ALSO returned the replacement" deep-cut

A signal Etsy actually cares about: customers who got a replacement and still returned it. That's the refund-the-relationship trigger. Self-join the returns table by original_order_id:

SELECT r2.customer_id
FROM returns r1
JOIN returns r2 ON r2.order_id = r1.replacement_order_id
   AND r2.customer_id = r1.customer_id;
The full segment definition — combine everything
-- "Frequent recent returners with high return-rate, excluding low-volume noise"
WITH last_90d AS (
  SELECT customer_id, COUNT(*) AS recent_returns
  FROM returns
  WHERE return_date >= CURRENT_DATE - INTERVAL '90 days'
  GROUP BY customer_id
),
volumes AS (
  SELECT customer_id, COUNT(*) AS orders_90d
  FROM orders
  WHERE order_date >= CURRENT_DATE - INTERVAL '90 days'
  GROUP BY customer_id
)
SELECT v.customer_id, v.orders_90d, COALESCE(l.recent_returns, 0) AS returns_90d,
  ROUND(100.0 * l.recent_returns / NULLIF(v.orders_90d, 0), 1) AS return_rate_90d
FROM volumes v
LEFT JOIN last_90d l USING (customer_id)
WHERE v.orders_90d >= 5
  AND COALESCE(l.recent_returns, 0) >= 3
ORDER BY return_rate_90d DESC;
Senior signal: answer the literal question first, then immediately surface the marketing-actionable variants — threshold, recency window, return-rate denominator, low-volume guard. The "ALSO returned the replacement" follow-up shows familiarity with how returns operations actually work.
