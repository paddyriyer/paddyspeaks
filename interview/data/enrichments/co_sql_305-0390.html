LinkedIn Recruiter's per-seat usage analysis: how many searches do recruiters run per day, and how many are active each month. The query uses window aggregates for cross-day totals.

2. The SQL — per-day searches + per-recruiter rollup

WITH per_day AS (
  SELECT recruiter_id, DATE(searched_at) AS day, COUNT(*) AS searches
  FROM recruiter_searches GROUP BY recruiter_id, DATE(searched_at)
)
SELECT recruiter_id, day, searches,
  SUM(searches) OVER (PARTITION BY recruiter_id) AS total_searches,
  COUNT(*) OVER (PARTITION BY recruiter_id) AS active_days
FROM per_day
ORDER BY recruiter_id, day;

3. Strategic explanations

The 2-CTE shape

Aggregate to (recruiter, day) first; then window-aggregate across days. Two passes, clear semantics. Cleaner than nested aggregates.

The "active recruiter" definition

LinkedIn defines an active Recruiter seat as: 5+ searches in 30 days. The active-days count is the input to renewal-prediction.

Per-seat ROI signal

Recruiter seat costs $9000+/year. Renewal depends on usage:

The intervention threshold

Recruiters with < 5 days/month get a Customer Success outreach. The query result feeds the alerting trigger.

Search-quality vs quantity

500 searches with no InMails sent ≠ 50 with high engagement. Production tracks downstream metrics. Search count is the activity signal; InMail-sent is the value signal.

Senior signal: use 2-CTE structure for per-grain-then-rollup, define "active" as 5+ searches/30 days, surface the seat-renewal probability ladder, and pair quantity with downstream-quality signals.
