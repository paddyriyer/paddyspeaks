An aviation data team needs a name-length leaderboard: "For each country, which airport has the longest name?" This combines a JOIN between airports and their countries, a string-length calculation, and a top-1-per-group selection using a window-function rank. The interviewer is checking the ranked-subquery pattern for per-partition topN, and whether the candidate knows the Snowflake QUALIFY shortcut that eliminates the subquery entirely.

2. The SQL — RANK per country on name length, filter to top 1

SELECT code, name, country_name, name_length FROM (
  SELECT a.code, a.name, c.country_name, LENGTH(a.name) AS name_length,
         RANK() OVER (PARTITION BY c.country_name ORDER BY LENGTH(a.name) DESC) AS rk
  FROM airports a
  JOIN countries c ON a.country_id = c.country_id
) t
WHERE rk = 1
ORDER BY country_name;

4. Strategic explanations

The top-1-per-group pattern — aggregate first, rank, then filter

The query follows the standard ranked-subquery template for per-group top-N: compute the rank inside a subquery or CTE, then filter on rk = 1 in the outer query. The window function RANK() OVER (PARTITION BY country_name ORDER BY LENGTH(name) DESC) assigns rank 1 to the airport(s) with the greatest name length within each country. The outer WHERE rk = 1 then keeps only the leader(s). This works because window functions are computed after GROUP BY but before WHERE in the outer query.

RANK vs ROW_NUMBER for ties

RANK assigns the same rank to tied rows. If two airports in the same country had identical name lengths (the maximum), both receive rk=1 and both appear in the output — a correct "co-winners" result. ROW_NUMBER would arbitrarily pick one, silently discarding the other. For a "longest name" query where ties should be visible, RANK is the right choice.

Snowflake QUALIFY shortcut — eliminating the subquery

The dialect_token for this question is QUALIFY, flagging that the Snowflake (and BigQuery) rewrite is:

SELECT a.code, a.name, c.country_name, LENGTH(a.name) AS name_length
FROM airports a
JOIN countries c ON a.country_id = c.country_id
QUALIFY RANK() OVER (PARTITION BY c.country_name ORDER BY LENGTH(a.name) DESC) = 1
ORDER BY country_name;

QUALIFY filters on window-function results without a wrapping subquery, much like HAVING filters on aggregate results. It is not part of the SQL standard and is not supported in Postgres, MySQL, or SQL Server — those engines require the subquery pattern.

LENGTH vs character_length vs len

LENGTH(name) counts characters in Postgres (and Snowflake, BigQuery). In Postgres with multi-byte UTF-8 strings, LENGTH counts characters (code points), while OCTET_LENGTH counts bytes. For airport names using only ASCII, the two are identical. In SQL Server, the equivalent is LEN(name) (which also counts characters and ignores trailing spaces). Always clarify whether "longest" means most characters or most bytes when dealing with non-ASCII data.

JOIN type — airports without a country are silently dropped

The INNER JOIN drops any airport whose country_id has no matching row in countries. For a well-maintained reference table this is acceptable. In a data warehouse where FK constraints are not enforced, it is worth running a data-quality check: SELECT COUNT(*) FROM airports WHERE country_id NOT IN (SELECT country_id FROM countries) to confirm no orphan airports exist before trusting the result.

Senior signal: articulate the ranked-subquery top-N pattern, choose RANK over ROW_NUMBER so ties are not silently discarded, demonstrate the Snowflake QUALIFY shortcut that eliminates the subquery wrapper, and flag that the INNER JOIN drops airports without a matching country — a data-quality risk in schema-on-read warehouses.
