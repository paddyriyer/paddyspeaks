Salesforce's marketing-funnel headline: lead → opportunity rate per source. Combined with avg time-to-convert, it tells which channels deliver real pipeline vs vanity volume.

2. The SQL — conversion + velocity per source

SELECT source, COUNT(*) AS leads, COUNT(converted_at) AS converted,
  ROUND(100.0 * COUNT(converted_at) / NULLIF(COUNT(*), 0), 1) AS conversion_pct,
  ROUND(AVG(CASE WHEN converted_at IS NOT NULL
    THEN (julianday(converted_at) - julianday(created_at)) END), 1) AS avg_days_to_convert
FROM leads GROUP BY source HAVING COUNT(*) >= 3 ORDER BY conversion_pct DESC;

3. Strategic explanations

The COUNT-of-non-NULL idiom

COUNT(converted_at) only counts non-NULL — the converted leads. Cleaner than CASE WHEN.

Quality vs volume

"Webinar" source: 50 leads × 30% conv × 5 days = high quality. "Bulk list buy": 5000 leads × 0.5% conv × 60 days = volume only. Conversion + days surface both.

The AVG with CASE — guard against converted=NULL skewing the mean

Without the CASE, AVG would compute (julianday(NULL) - julianday(...)) — undefined. The CASE forces NULL on unconverted rows so AVG ignores them.

The marketing-attribution rabbit hole

"Source" is single-touch. Real B2B journeys touch 7-10 channels. Multi-touch attribution (linear, time-decay, U-shape) gives different per-source credit. This query is the first-touch baseline; production runs all four models.

Senior signal: COUNT-of-non-NULL for sequential gates, AVG-with-CASE to exclude unconverted, distinguish quality (rate × velocity) from volume, and reframe single-touch source as the first of multiple attribution models.
