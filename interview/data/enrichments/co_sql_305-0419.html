TikTok's Duet and Stitch features create derivative content — videos responding to or remixing originals. High derivative count = original video achieved memetic status.

2. The SQL — original to derivatives self-join

SELECT v.video_id AS original_video,
  COUNT(d.video_id) AS derivatives,
  SUM(CASE WHEN d.video_type = 'duet'   THEN 1 ELSE 0 END) AS duets,
  SUM(CASE WHEN d.video_type = 'stitch' THEN 1 ELSE 0 END) AS stitches
FROM videos v
LEFT JOIN videos d ON d.parent_video_id = v.video_id
WHERE v.parent_video_id IS NULL
GROUP BY v.video_id ORDER BY derivatives DESC LIMIT 20;

3. Strategic explanations

The self-join shape

Same table, two aliases: v for originals (parent_video_id IS NULL), d for derivatives (parent_video_id = v.video_id). LEFT JOIN keeps originals with zero derivatives.

Duet vs Stitch — different mechanics
typestructuretypical use
Duetside-by-side, plays simultaneouslyreaction, harmony
Stitch5-sec clip from original, then own content"my response", commentary
Memetic threshold

100+ derivatives = meme. Top originals reach 10K+. The query result is the input to "trending challenges" feature.

Permission-revoke handling

Original creator can disable Duet/Stitch retroactively. Existing derivatives stay; new ones blocked. Production has a derivatives_disabled_at column.

Cross-platform leak (the dark side)

Derivatives uploaded to Instagram Reels reduce TikTok's value capture. Production tracks "derivative-only" creators who drive TikTok engagement but monetize elsewhere.

Senior signal: self-join with LEFT to keep zero-derivative originals, contrast Duet (parallel) vs Stitch (sequential) mechanics, identify 100+ derivatives as meme threshold, and acknowledge cross-platform leak risk.
