Facebook Groups follow the classic 1-9-90 rule: 1% post, 9% comment, 90% lurk (or react). Group-Health team monitors the pyramid because top-of-funnel posters drive the entire group's vitality. If posters fall to zero, the group dies in 30 days. The query classifies users by their highest-engagement action and counts each tier.

2. The SQL — multi-level CASE classification

WITH per_user AS (
  SELECT group_id, user_id,
    SUM(CASE WHEN action_type = 'post'    THEN 1 ELSE 0 END) AS posts,
    SUM(CASE WHEN action_type = 'comment' THEN 1 ELSE 0 END) AS comments,
    SUM(CASE WHEN action_type = 'reaction' THEN 1 ELSE 0 END) AS reactions
  FROM group_actions
  GROUP BY group_id, user_id
)
SELECT group_id,
  SUM(CASE WHEN posts >= 1                THEN 1 ELSE 0 END) AS posters,
  SUM(CASE WHEN posts = 0 AND comments >= 1 THEN 1 ELSE 0 END) AS commenters,
  SUM(CASE WHEN posts = 0 AND comments = 0 AND reactions >= 1 THEN 1 ELSE 0 END) AS reactors,
  COUNT(DISTINCT user_id) AS active_members
FROM per_user
GROUP BY group_id
ORDER BY active_members DESC;

3. Strategic explanations

The "highest action" classification — mutually exclusive

A user who both posts AND comments should count once as a "poster", not twice. The cascading posts = 0 AND ... predicates enforce mutual exclusion. Without that, summing the three columns would exceed active_members — clearly wrong.

The "poster churn" warning

Track posters week-over-week. Top-5% groups have 80% poster retention; declining groups dip to 30% — that's the early-warning signal 4-8 weeks before the group flatlines. Pair this query with a LAG-based week-over-week diff to catch the slope.

The "lurker activation" intervention

The product team's actual question: "can we move 10% of reactors into commenters?". That requires a different CTE: which reactors have stopped reacting (engagement-decline early) — they're the easiest to reactivate. Different query, related framework.

The action-quality bonus

"Posts" isn't a single thing. A 2-paragraph thoughtful post and a "lol" share are both action_type = 'post'. Production splits posts by quality (length, image included, prompt-followed) for the actual influence ranking. Mention this when probed.

The community-manager signal

Most groups have 1-3 "anchor" posters who drive 50%+ of content. Identifying them and protecting their engagement is the highest-leverage Group-Health intervention. The query above doesn't rank posters within a group — add a window function to identify top contributors:

RANK() OVER (PARTITION BY group_id ORDER BY posts DESC) AS poster_rank
Senior signal: use cascading CASE for mutually-exclusive tier classification, name the 1-9-90 expectation as the diagnostic baseline, and pre-empt the "poster churn" early-warning use case. The anchor-poster identification follow-up (RANK on posts) is the over-the-bar move.
