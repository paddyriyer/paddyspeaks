Meta uses "Surrounded Regions" to test a counter-intuitive framing. A board of 'X' and 'O'; any region of 'O's fully enclosed by 'X' should be flipped to 'X'. The trap is to hunt for enclosed regions directly — deciding "is this blob surrounded?" is fiddly. The senior move: flip the question and find the uncapturable 'O's instead, which are exactly those connected to a border.

1. The input data

A 4×4 board. 'O's touching the edge survive; interior 'O's get captured:

board = [
  ['X','X','X','X'],
  ['X','O','O','X'],
  ['X','X','O','X'],
  ['X','O','X','X'],
]
# the O at (3,1) touches the bottom edge -> safe
# the O-blob at (1,1)(1,2)(2,2) is fully enclosed -> captured

2. The approach

An invert-the-problem flood fill in three phases.

  1. Mark survivors. Run dfs from every 'O' on the four borders — the loops call dfs(r,0), dfs(r,C-1), dfs(0,c), dfs(R-1,c). The DFS rewrites every border-connected 'O' to a sentinel '#'. These cells cannot be captured because they (or their blob) leak off the edge.
  2. Decide. Now the board has three symbols. After the sweep, any remaining 'O' must have been unreachable from the border — i.e. genuinely enclosed.
  3. Rewrite. The final double loop sets each cell: '#' → 'O' (restore the survivors) and everything else → 'X' (capture the enclosed 'O's, and harmlessly re-set the existing 'X's).

The '#' sentinel is what lets one boolean — "did the flood reach you?" — be stored directly in the board. The function mutates board in place and returns nothing.

The enclosed blob at (1,1), (1,2), (2,2) is flipped to X. The O at (3,1) touches the bottom border, so it survives.

4. Strategic explanations

Complexity

Time O(R·C) — each border DFS sinks cells to '#' so no cell is visited twice, and the final rewrite is one more linear sweep. Space O(R·C) worst case for recursion depth when the board is one large border-connected 'O' region.

The idiomatic alternative

An iterative BFS with collections.deque, seeded with every border 'O' at once, avoids recursion-limit risk:

from collections import deque
q = deque((r,c) for r in range(R) for c in range(C)
          if (r in (0,R-1) or c in (0,C-1)) and board[r][c]=='O')
while q:
    r,c = q.popleft(); board[r][c]='#'
    for nr,nc in ((r+1,c),(r-1,c),(r,c+1),(r,c-1)):
        if 0<=nr<R and 0<=nc<C and board[nr][nc]=='O':
            q.append((nr,nc))

Same O(R·C); the queue lives on the heap.

The edge case

if not board guards an empty input. A 1-row or 1-column board: every cell is a border cell, so no 'O' can ever be enclosed — all 'O's survive. A board with no 'O's → the DFS never fires, the rewrite leaves it unchanged. An 'O' blob that touches the edge at just one cell saves the entire blob — connectivity, not per-cell adjacency, decides survival.

The follow-up

"Diagonal connectivity counts" → add four diagonal deltas. "Don't mutate the input" → replace the '#' sentinel with a safe set of coordinates and consult it in the rewrite. "Return how many 'O's were captured" → count during the final sweep.

The trap

The losing approach: DFS from each interior 'O' and try to prove its blob never touches an edge. It works but is far messier — you must track edge-contact through the recursion and re-scan blobs. Worse is the off-by-one trap: flipping all 'O's to 'X' first and then trying to "un-flip" border ones — you've already lost the information about which were connected. Mark survivors before you flip anything.

Senior signal: inverts the problem — floods inward from the border to mark the uncapturable 'O's rather than hunting enclosed regions directly — uses a '#' sentinel to store the reachability bit in the board itself, gets the three-state final rewrite right, and offers the iterative BFS when the recursion stack is a worry.
