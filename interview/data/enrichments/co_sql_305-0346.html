Amazon's catalog is the largest in commerce — billions of ASINs, dozens of marketplaces, constant updates. Late-arriving dimensions (ASIN metadata appearing AFTER the order events) are routine. The query LEFT JOINs orders to the most-recent ASIN dim row + COALESCE placeholder.

2. The SQL — LAD pattern

WITH current_dim AS (
  SELECT asin, title, category,
    ROW_NUMBER() OVER (PARTITION BY asin ORDER BY effective_date DESC) AS rn
  FROM asin_dim
)
SELECT o.order_id, o.asin,
  COALESCE(d.title,    'ASIN ' || o.asin || ' (Pending Catalog Sync)') AS title,
  COALESCE(d.category, 'Uncategorized') AS category,
  o.ordered_at
FROM orders o
LEFT JOIN current_dim d ON d.asin = o.asin AND d.rn = 1
ORDER BY o.ordered_at DESC;

3. Strategic explanations

Why LEFT JOIN, not INNER

An inner join silently DROPS orders whose ASIN dim hasn't propagated yet. With Amazon's catalog scale, that means losing 0.1-0.5% of orders during normal operations and 5%+ during a Prime Day catalog refresh. LEFT JOIN keeps every order; the COALESCE handles missing dim.

Most-recent dim row via ROW_NUMBER

SCD2 dims have multiple rows per ASIN (re-categorizations). ROW_NUMBER() OVER (PARTITION BY asin ORDER BY effective_date DESC) + WHERE rn = 1 picks the current one. Critical for re-titled or re-genre'd products.

The placeholder string

"Pending Catalog Sync" is searchable — the BI tool's filter dropdown shows it as a real bucket. The analyst can WHERE category = 'Uncategorized' to find all orphans. Plain NULL would be invisible.

approachwhere the COALESCE lives
At-ingestETL inserts a placeholder dim row when fact arrives without dim

At-ingest is cleaner for downstream analytics; at-query is loosely coupled. Production lakehouses prefer at-ingest.

Late-arriving FACT (the dual problem)

The opposite case: dim is current, fact arrives days late from offline mobile. Should the late fact attribute to the dim that was current AT FACT TIME or NOW? SCD2 + temporal join solves it:

JOIN asin_dim d ON d.asin = o.asin
  AND o.ordered_at BETWEEN d.effective_from AND COALESCE(d.effective_to, '9999-12-31')
The orphan-rate alert
SELECT 100.0 * SUM(CASE WHEN d.title IS NULL THEN 1 ELSE 0 END) / COUNT(*) AS pct_orphan
FROM orders o LEFT JOIN current_dim d ON d.asin = o.asin AND d.rn = 1;

Production target < 0.5%. Above 1% pages the data-platform on-call.

Senior signal: name LEFT JOIN + COALESCE + ROW_NUMBER as the LAD trio, contrast at-ingest vs at-query inferred-member strategies, raise the late-arriving-fact temporal-join cousin, and pin the orphan-rate SLO.
