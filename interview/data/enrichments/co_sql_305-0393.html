LinkedIn's content team compares engagement rates across post formats: article / update / video / poll. The query produces engagement rate (engagements / impressions) per format.

2. The SQL — per-format engagement rate

SELECT post_type,
  COUNT(*) AS posts,
  SUM(impressions) AS total_impressions,
  SUM(engagements) AS total_engagements,
  ROUND(100.0 * SUM(engagements) / NULLIF(SUM(impressions), 0), 2) AS engagement_rate_pct,
  ROUND(SUM(engagements) * 1.0 / NULLIF(COUNT(*), 0), 1) AS avg_engagements_per_post
FROM posts GROUP BY post_type
ORDER BY engagement_rate_pct DESC;

3. Strategic explanations

Engagement rate vs avg-per-post

Two complementary metrics:

  • Rate (eng/impressions) — quality of viewer reach
  • Avg per post — absolute scale

A format with 5% rate × 1K impressions = 50 engagements per post. Rate-only would prefer it; avg-per-post-only might prefer a 1% × 1M format.

Polls win on rate because the action is built-in (clicking is engaging).

The content-team trade-off

Articles have lower rate but produce SEO + thought-leadership benefits beyond engagement. Optimizing only for rate would kill long-form. Production reports both rate AND "downstream" metrics (profile views from articles).

The algorithm-amplification feedback

LinkedIn's feed promotes high-engagement formats more. High polls rate → more polls served → more polls posted → market saturation. Production includes "freshness" features to prevent any format from dominating.
