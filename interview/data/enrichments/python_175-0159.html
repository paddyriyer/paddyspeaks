Hierholzer's algorithm for an Eulerian path. The same shape as DNA fragment assembly (Illumina's de Bruijn graph reconstruction), chip routing through net junctions (Cadence), garbage-collection epoch traversal of root-set edges, and any "use every edge exactly once" workflow. The trap most candidates fall into is greedy lex-DFS: at JFK they go to the lex-smallest neighbour, dead-end, and have no way to back out. Hierholzer's insight is that you must go past the dead end first — descend until you have no edges left to use, append the dead-end node, then unwind. The post-order append, reversed at the end, gives the path. Visa asks this version because their payment-routing graph traversal — visiting every link in a card-issuer-to-acquirer network exactly once during reconciliation — is structurally identical.

1. The problem in one sentence + the key insight

Given a list of one-way airline tickets, reconstruct an itinerary that starts at JFK and uses every ticket exactly once. If multiple valid itineraries exist, return the lexicographically smallest. Insight: this is finding an Eulerian path in a directed multigraph. Hierholzer's algorithm: from the start node, DFS greedily (using a min-heap to break ties lex-smallest), but only append a node to the route AFTER all its outgoing edges have been consumed. Reverse the post-order list at the end. The greedy walk may dead-end at the wrong node, but the post-order append "undoes" that — the dead-end node becomes the LAST element of the final reversed route.

2. Worked example — the classic LC trap case

Tickets: [[JFK,SFO], [JFK,ATL], [SFO,ATL], [ATL,JFK], [ATL,SFO]]. The greedy "always take lex-smallest first" produces JFK → ATL → JFK → SFO → ATL → SFO. Trace Hierholzer's:

Adjacency (min-heap per source, all sorted lex):
  JFK: [ATL, SFO]
  ATL: [JFK, SFO]
  SFO: [ATL]

route = []
dfs(JFK):
  pop ATL from JFK.heap   -> JFK: [SFO]
  dfs(ATL):
    pop JFK from ATL.heap -> ATL: [SFO]
    dfs(JFK):
      pop SFO from JFK.heap -> JFK: []
      dfs(SFO):
        pop ATL from SFO.heap -> SFO: []
        dfs(ATL):
          pop SFO from ATL.heap -> ATL: []
          dfs(SFO):
            no edges; route.append(SFO)         route=[SFO]
          route.append(ATL)                     route=[SFO, ATL]
        route.append(SFO)                       route=[SFO, ATL, SFO]
      route.append(JFK)                         route=[SFO, ATL, SFO, JFK]
    route.append(ATL)                           route=[SFO, ATL, SFO, JFK, ATL]
  route.append(JFK)                             route=[SFO, ATL, SFO, JFK, ATL, JFK]

return route reversed:  [JFK, ATL, JFK, SFO, ATL, SFO]

The critical moment is when we're at JFK with one ticket left (JFK→SFO) and ATL still has tickets out. Naive greedy would have gone JFK→ATL first and then dead-ended at SFO with ATL's tickets stranded. Hierholzer's instead recurses into the dead-end branch first, appends, then unwinds — so the dead-end winds up at the END of the route, where it belongs.

3. The algorithm — code with inline comments

from collections import defaultdict
import heapq

def findItinerary(tickets):
    # Adjacency: source -> min-heap of destinations. Min-heap gives O(log d) pop
    # of the lex-smallest unused edge.
    g = defaultdict(list)
    for a, b in sorted(tickets): heapq.heappush(g[a], b)

    route = []  # POST-ORDER append list. Reverse at the end.
    def dfs(u):
        # Consume edges greedily, ALWAYS lex-smallest first.
        while g[u]: dfs(heapq.heappop(g[u]))
        # Only after no outgoing edges remain do we record this node.
        route.append(u)

    dfs('JFK')
    return route[::-1]

Data structures: defaultdict(list) mapping source → min-heap (via heapq), and a single post-order route list. Invariant: when dfs(u) returns, every outgoing edge from u (and transitively every reachable edge) has been consumed. The post-order property guarantees that when reversed, the route uses each ticket exactly once.

4. Complexity

Time: O(E log E) — each of the E tickets is heappushed and heappopped once. Space: O(V + E) for the graph + O(V) recursion depth. The sorted(tickets) + per-source heap construction is itself O(E log E); a small optimisation is to skip the initial sort and let the heap order things, but the constant is the same.

5. The interviewer's probes

Probe 1 — Degenerate input: a single ticket. [[JFK, A]]. dfs(JFK) pops A, dfs(A) finds empty heap, appends A; back in dfs(JFK), appends JFK. Reverse: [JFK, A]. The base case is just "no outgoing edges → append".

Probe 2 — Cycle back to JFK. [[JFK,A],[A,JFK],[JFK,B]]. The algorithm goes JFK→A→JFK→B, which is correct: each edge consumed exactly once, B is the dead end and appears last. This is the case where "lex-smallest greedy" works coincidentally; the post-order is what makes the harder cases correct.

Probe 3 — Generalisation: what if multiple JFK→ X tickets exist? The min-heap handles duplicates naturally — two JFK→A tickets mean A is popped twice from JFK's heap. The graph is a multigraph; the heap just stores destinations with repetition.

Probe 4 — What if no Eulerian path exists? The problem statement guarantees one does, but the algorithm doesn't detect it. To check: Eulerian path exists iff (a) the graph is connected (restricted to nodes with edges) and (b) exactly zero or two nodes have in_degree ≠ out_degree. Naming this check is a senior detail.

Probe 5 — Performance trap: recursion depth. For 10,000 tickets in a single long chain, recursion depth = 10,000, which overflows Python's default limit (1000). Either sys.setrecursionlimit(20000) or convert to an iterative explicit-stack DFS. The iterative version is the textbook Hierholzer.

Probe 6 — Alternative algorithm: Fleury's algorithm. Fleury picks edges one-at-a-time, never crossing a "bridge" until forced. O(E²) due to per-step bridge detection. Strictly worse than Hierholzer for this problem, but the comparison shows you know the literature.

6. Where this pattern appears in production

  • DNA fragment assembly (Illumina sequencers, SPAdes) — a de Bruijn graph's Eulerian path reconstructs the original genome from k-mer fragments. The dead-end-first walk is identical; the substrate is genomic k-mers instead of airport codes.
  • Chip routing (Cadence Allegro, KiCad's freerouter) — a router must traverse every net exactly once during a continuity check; Hierholzer's algorithm validates routability.
  • Postal / delivery route planning (Chinese Postman variant) — visit every street exactly once, return to depot. Pure Eulerian for connected even-degree graphs; Hierholzer is the algorithm.
  • De Bruijn sequence generation (cryptographic key schedules, magnetic-stripe brute force) — the shortest string containing every length-k sequence exactly once is a Eulerian path on a (k-1)-mer graph; Hierholzer constructs it.
  • Payment network reconciliation (Visa, Mastercard, SWIFT) — visiting every issuer-to-acquirer link in a daily settlement graph exactly once for audit trail construction; same algorithm, multi-billion-dollar substrate.
