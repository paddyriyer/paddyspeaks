Regional subtotals with a grand-total row are a classic reporting requirement. The complication this question injects is that GROUP BY ROLLUP signals its summary rows with NULL in the grouped columns — identical to a real NULL region value in production data. The interviewer is probing whether you know the GROUPING() function, which distinguishes "NULL because it's a ROLLUP summary" from "NULL because the data is missing".

1. The source data — sales

regionproductamount
NorthA100
NorthB50
SouthA70
SouthC30
EastB40

Three regions, totals: North 150, South 100, East 40, grand total 290. The product column is not grouped — ROLLUP(region) only introduces one extra summary level (the grand total), not per-product subtotals.

2. The SQL — GROUP BY ROLLUP with GROUPING() to label the grand total

SELECT
  CASE WHEN GROUPING(region) = 1 THEN 'Grand Total' ELSE region END AS region,
  SUM(amount) AS total
FROM sales
GROUP BY ROLLUP (region)
ORDER BY GROUPING(region), region;

ROLLUP(region) produces one row per distinct region plus one extra super-aggregate row. The GROUPING(region) function returns 1 for the super-aggregate row and 0 for normal rows, enabling the CASE to inject the label 'Grand Total' in place of NULL. The ORDER BY GROUPING(region), region sorts normal regions alphabetically first, then the grand total last.

3. The resulting analysis

regiontotal
East40
North150
South100
Grand Total290

East: 40. North: 100+50=150. South: 70+30=100. Grand Total: 40+150+100=290. The grand total row sorts last because GROUPING(region) = 1 sorts after 0. The three region rows sort alphabetically within the 0-grouping partition.

4. Strategic explanations

How ROLLUP works

GROUP BY ROLLUP(region) is shorthand for GROUP BY GROUPING SETS ((region), ()). It produces one group per distinct region value (the normal aggregation) plus one group for the empty set () — the grand total. The empty-set group has NULL in the region column of the output. With two dimensions ROLLUP(region, product) would add a subtotal level per region across products, and then the grand total — three levels in all.

GROUPING() vs IS NULL for distinguishing summary rows

A naive CASE WHEN region IS NULL THEN 'Grand Total' ELSE region END misidentifies real NULL regions as summary rows. GROUPING(region) returns a bit-flag: 1 means "this column was collapsed in a ROLLUP/CUBE summary", 0 means it is a real group value (which may itself be NULL). This distinction is the entire point of the question; the wrong answer is functionally correct only when your data can never have a NULL region, which is an unsafe assumption to build a report on.

ORDER BY with GROUPING() to push grand total last

ORDER BY GROUPING(region), region works because GROUPING() is allowed in the ORDER BY clause (it evaluates on the already-grouped result set). The value is 0 for all real groups and 1 for the grand total, so they sort in that order. An alternative is ORDER BY CASE WHEN GROUPING(region) = 1 THEN 1 ELSE 0 END, region — identical in effect. Adding NULLS LAST to region is redundant here because GROUPING(region) = 1 already sends that row to the bottom, but it would matter if real NULL regions existed.

CUBE and GROUPING SETS for multi-dimensional reports

CUBE(region, product) adds subtotals for every subset of dimensions; GROUPING SETS((region),(product),(region,product),()) lets you choose exactly which combinations. For this question — one dimension, one grand total — ROLLUP is the minimal correct tool. Demonstrating awareness of the full GROUPING SETS family shows you understand the generalisation.

Dialect notes

GROUP BY ROLLUP and GROUPING() are part of SQL:1999 and are supported by PostgreSQL, Oracle, SQL Server, DB2, Snowflake, BigQuery, and MySQL 8.0+. SQLite does not support ROLLUP. In older MySQL (5.x) ROLLUP syntax differs slightly: GROUP BY region WITH ROLLUP, and GROUPING() was added in MySQL 8.0.

Senior signal: use GROUPING(col) = 1 — not col IS NULL — to detect ROLLUP summary rows, because production data may contain legitimate NULLs that would be misclassified; sort the grand total last with ORDER BY GROUPING(region), region; and know that ROLLUP is syntactic sugar for GROUPING SETS((region), ()), which generalises to CUBE and arbitrary GROUPING SETS for multi-dimensional reports.
