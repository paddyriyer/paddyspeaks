Spotify's path-to-paid centers on the 30-day Premium trial. Country-level conversion divergence (50% in US, 12% in IN) maps to local affordability + payment-method coverage.

2. The SQL — per-country conversion + cancel

SELECT country,
  COUNT(*) AS trials,
  COUNT(converted_to_paid) AS converted,
  ROUND(100.0 * COUNT(converted_to_paid) / NULLIF(COUNT(*), 0), 1) AS conversion_pct,
  COUNT(CASE WHEN cancelled_at IS NOT NULL THEN 1 END) AS cancelled
FROM trials
GROUP BY country
HAVING COUNT(*) >= 3
ORDER BY conversion_pct DESC;

3. Strategic explanations

The COUNT(col) idiom

COUNT(converted_to_paid) counts non-NULL values — a clean way to count "converted" without CASE. Sequential funnel gates use this pattern repeatedly.

The local-affordability breakdown

Premium-monthly varies: $9.99 US, ₹119 India, R$ 19.90 Brazil. Conversion rate is sensitive to local-monthly-as-% of GDP. Production overlays affordability in the same chart.

The payment-method coverage

India's primary payment is UPI; if Spotify hasn't enabled UPI in market X, conversion suffers regardless of affordability. Always join to country_payment_method_coverage before declaring "country X is hard".

Family / Duo plan offset

Some markets prefer Family ($14.99 / 6 users) over individual. Splitting "converted" by which plan reveals: India might convert at 12% individual but 22% family.

In-flight cohort exclusion

Cohorts that signed up in the last 30 days haven't completed trial yet. Filter:

WHERE julianday(trial_start) <= julianday('2024-04-01') - 30
Senior signal: use COUNT(col) for non-NULL gates, overlay affordability + payment-coverage to explain country variance, split by plan type (individual vs family), and exclude in-flight cohorts.
