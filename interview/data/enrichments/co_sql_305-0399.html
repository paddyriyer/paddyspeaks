Pinterest's defining action: save. Save-rate per category measures content resonance. Wedding pins save at 8-15% — extremely high; news pins save at 0.5%.

2. The SQL — save rate per category

SELECT category,
  SUM(CASE WHEN action_type = 'view' THEN 1 ELSE 0 END) AS views,
  SUM(CASE WHEN action_type = 'save' THEN 1 ELSE 0 END) AS saves,
  ROUND(100.0 * SUM(CASE WHEN action_type = 'save' THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN action_type = 'view' THEN 1 ELSE 0 END), 0), 2) AS save_rate_pct
FROM pin_events GROUP BY category
HAVING SUM(CASE WHEN action_type = 'view' THEN 1 ELSE 0 END) >= 3
ORDER BY save_rate_pct DESC;

The "save without click" cliff

Many users save without clicking through (saves directly from the feed). That's the ideal: discovery without leaving the platform. Track click-after-save for the "did the saver actually act" funnel.

Seasonality

Wedding pins peak Jan-Mar (wedding planning). Holiday recipes peak Oct-Dec. The category-baseline must control for season.

Idea Pin vs Standard Pin

Idea Pins (multi-page) have 2-3x higher save rate but fewer impressions. Production splits.

Senior signal: save as highest-intent action, cite category-specific baselines, distinguish save-without-click as the discovery-without-leaving signal, control for seasonality, and split Idea Pin vs Standard.
