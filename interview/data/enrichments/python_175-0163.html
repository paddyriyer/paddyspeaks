Topological sort over a constraint graph. The same shape as build-system task ordering (Bazel, npm, make), dependency resolution (apt, pip), schema migration ordering, and any "derive a total order from pairwise constraints" workflow. The senior insight here is that the input is NOT a graph — it's a list of words. The hard part is deriving the graph: pairwise compare adjacent words, find the first differing character, and emit one directed edge. Once you have the graph, Kahn's BFS (or DFS + reverse post-order) gives the order in O(V + E). Bloomberg asks this version because their terminal's symbol-tickering subsystem builds intra-day a fresh ordering of related instruments from pairwise correlation evidence — same algorithm, same graph extraction step from a stream of observations.

1. The problem in one sentence + the key insight

You're given words from an alien language in sorted order; derive the alphabet order. Insight: for each adjacent pair (w[i], w[i+1]), the first character at which they differ tells you exactly one precedence edge (e.g. "wrf" < "er" gives w < e); identical-prefix cases give NO new information; and a longer-prefix-followed-by-shorter (e.g. "abc" before "ab") is INVALID — return "". Once the graph is built, Kahn's algorithm (BFS over in-degree-0 nodes) produces a valid topological order, or detects a cycle if the final order length is less than the node count.

2. Worked example — words ['wrt','wrf','er','ett','rftt'], derive 'wertf'

Step 1 — Initialise in-degrees of every letter seen:
  chars = {w, r, t, f, e}
  deg = {w:0, r:0, t:0, f:0, e:0}

Step 2 — Scan adjacent pairs, derive one edge per pair:

  ('wrt', 'wrf')  -> first differing char: t vs f at index 2  -> edge t -> f
                                                                   deg[f] = 1
  ('wrf', 'er')   -> first differing char: w vs e at index 0  -> edge w -> e
                                                                   deg[e] = 1
  ('er',  'ett')  -> first differing char: r vs t at index 1  -> edge r -> t
                                                                   deg[t] = 1
  ('ett', 'rftt') -> first differing char: e vs r at index 0  -> edge e -> r
                                                                   deg[r] = 1

Adjacency graph:
  w -> e -> r -> t -> f
deg = {w:0, e:1, r:1, t:1, f:1}

Step 3 — Kahn's BFS from in-degree-0 nodes:

  queue = [w]                    (only w has deg=0)
  pop w  -> order = [w]
          decrement deg[e]: 1-1=0  -> enqueue e
  pop e  -> order = [w, e]
          decrement deg[r]: 1-1=0  -> enqueue r
  pop r  -> order = [w, e, r]
          decrement deg[t]: 1-1=0  -> enqueue t
  pop t  -> order = [w, e, r, t]
          decrement deg[f]: 1-1=0  -> enqueue f
  pop f  -> order = [w, e, r, t, f]

len(order) == len(deg)  -> return 'wertf'

The trace shows the two halves clearly: pair-by-pair edge extraction is the data preparation; Kahn's BFS is the algorithm. The output 'wertf' is uniquely determined here because the constraint graph is a straight chain; in graphs with branches (e.g. two independent in-degree-0 nodes), any topological order is acceptable.

3. The algorithm — code with inline comments

from collections import defaultdict, deque

def alienOrder(words):
    # 1) Register every distinct character. The in-degree dict doubles as the
    # set of nodes; we don't have a separate "chars seen" structure.
    deg = {c: 0 for w in words for c in w}
    g = defaultdict(set)         # adjacency, deduped via set

    # 2) For each adjacent pair, derive at most one edge.
    for a, b in zip(words, words[1:]):
        # Prefix violation: a is longer than b but b is a prefix of a.
        if len(a) > len(b) and a.startswith(b): return ""
        for x, y in zip(a, b):
            if x != y:
                if y not in g[x]:        # dedupe edges (multi-pair x->y)
                    g[x].add(y)
                    deg[y] += 1
                break                    # only the FIRST diff matters

    # 3) Kahn's algorithm: BFS over in-degree-0 nodes.
    q = deque([c for c in deg if deg[c] == 0])
    order = []
    while q:
        c = q.popleft(); order.append(c)
        for nxt in g[c]:
            deg[nxt] -= 1
            if deg[nxt] == 0: q.append(nxt)

    # Cycle detection: if not all nodes made it into the order, a cycle exists.
    return ''.join(order) if len(order) == len(deg) else ""

Data structures: defaultdict(set) for adjacency (set dedupes redundant edges), an in-degree dict, and a deque for Kahn's BFS. Invariants: (1) deg[c] equals the number of un-processed predecessors of c; (2) the BFS only enqueues a node when ALL its predecessors have been finalised, so the output respects every constraint.

4. Complexity

Time: O(C) where C is the total number of characters across all words. The pair-scan is O(C); Kahn's BFS over the resulting graph is O(V + E), and V + E ≤ C. Space: O(V + E) for the graph and the queue. The 26-letter alphabet caps V at 26, but the algorithm is written for arbitrary alphabets — useful when generalising.

5. The interviewer's probes

Probe 1 — Empty / single word. Single word means no pairs, no edges; every char has deg=0, all get enqueued, output is any permutation of the unique chars. The algorithm returns ONE valid answer; the problem accepts any.

Probe 2 — Prefix violation: ['abc', 'ab']. 'abc' before 'ab' is impossible in any valid sort because 'ab' is a proper prefix of 'abc' and must come first in any prefix order. The len(a) > len(b) and a.startswith(b) check catches this and returns "". Junior trap: forgetting to check this — you'd return a valid-looking answer that secretly satisfies no constraint.

Probe 3 — Cycle: ['a', 'b', 'a']. Derives a<b and b<a; both deg become 1 and never reach 0. Kahn's BFS produces a length-0 order, then the len(order) != len(deg) guard returns "". Cycle detection is FREE in Kahn's algorithm; in DFS-based topo sort you'd need an explicit "gray" set.

Probe 4 — Multiple valid orders. Two disconnected components or two in-degree-0 nodes mean multiple valid orders. The problem accepts any one. If the interviewer wants the lex-smallest, replace deque with a heapq — same algorithm, O(log V) per op instead of O(1).

Probe 5 — DFS-based alternative. Three-colour DFS (white/grey/black): visit each unvisited node, push to a post-order stack, reverse at end. Same O(V+E), different cycle detection (re-visiting a grey node). Useful when you also need the cycle path itself for error reporting.

Probe 6 — Generalisation: derive a partial order, not a total one. The output above is a linear extension of a partial order. If you want the partial order itself (the Hasse diagram), output the transitive reduction of the edge set — used in dependency-graph visualisation tools.

6. Where this pattern appears in production

  • Build systems (Bazel, Buck, make, ninja) — derive task execution order from BUILD files. Each rule's deps attribute is an edge in the constraint graph; Kahn's algorithm schedules a valid order (and parallel groups by depth).
  • Package managers (apt, pip, npm) — install dependencies before dependents. Cycles in the graph (pip's circular-dep error) are detected via the same length-mismatch check.
  • Database schema migrations (Alembic, Flyway, Liquibase) — apply migrations in dependency order. The "down_revision" field is the back-edge; the dispatcher does Kahn's BFS over the migration DAG.
  • Compilers (LLVM pass manager, GCC) — optimisation passes have pairwise ordering constraints (loop-invariant code motion before strength reduction); the pass manager schedules them topologically.
  • Spark / Airflow DAG executors — every job graph IS a topological sort; the scheduler enqueues tasks whose upstream deps are all done. Same Kahn's-BFS skeleton, just with concurrency and retries layered on.
  • Bloomberg terminal symbol resolution — pairwise observed price/volume correlation between instruments is converted into a precedence graph for intraday rebalancing; the topological extension is what the trader sees.
