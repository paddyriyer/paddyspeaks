Meta's News Feed team optimizes one quantity above all: time spent per post type. The query slices total dwell across video / image / text / link / poll posts and computes each type's share of total feed time. The senior framing: dwell time is a proxy, not the goal — passive scrolling counts as engaged time even when nothing meaningful happens.

2. The SQL — per-type aggregation + window-aggregate share

SELECT post_type,
  COUNT(*) AS views,
  COUNT(DISTINCT user_id) AS unique_viewers,
  ROUND(AVG(dwell_seconds), 1) AS avg_dwell,
  ROUND(SUM(dwell_seconds) * 100.0
      / SUM(SUM(dwell_seconds)) OVER (), 1) AS share_of_total_time_pct
FROM feed_views
GROUP BY post_type
ORDER BY share_of_total_time_pct DESC;

3. Strategic explanations

The SUM(SUM()) OVER () aggregate-of-aggregate

Same idiom as Okta's percent-contribution: inner SUM is per post_type, outer SUM-window is across all post_types. One pass, no subquery. Critical for share calculations.

Dwell vs engagement — the senior reframe

"Time spent" is correlated with — but not the same as — engagement. A 30-second video autoplays while the user scrolls past = 30 seconds of dwell, zero meaningful engagement. Production metrics distinguish:

  • Dwell — total seconds visible
  • Active dwell — seconds with eye-tracking / scroll-stopped / hover
  • Meaningful interaction — like / comment / share / save

The query above measures dwell only. Senior interviewers will probe: "how would you measure if dwell is the right metric?".

Reach vs depth — multi-axis

views = total impressions. unique_viewers = distinct user reach. Their ratio is "views per viewer" — a depth metric. A post type with high views/viewer ratio is binge-worthy; low ratio is single-glance content. Always emit both.

The Reels squeeze

Reels (short video) launch in 2020 broke this analysis: a Reel is technically a "video" post_type but watches like a different beast — autoplay, vertical, swipeable. Production splits Reels into their own type before this query. Lumping them with regular video hides the impact.

The user-segment cross-tab

"Image dominates dwell" globally might mean "image dominates dwell among older users; video dominates among younger". The PM-actionable view splits by age band:

GROUP BY post_type, CASE WHEN age < 25 THEN 'gen_z' WHEN age < 40 THEN 'millennial' ELSE 'older' END
The ranker-loop — what this query feeds

Feed-ranking models use post-type dwell as a feature. The query result isn't just a dashboard — it retrains the ranker weekly. A drift in average dwell for "video" might mean the ranker is over-weighting video; a/b test the response.

Senior signal: name the SUM-of-SUM aggregate-of-aggregate, contrast dwell vs active dwell vs meaningful interaction, and emit reach + depth simultaneously. The Reels-vs-video-lumping trap is the over-the-bar deep cut.
