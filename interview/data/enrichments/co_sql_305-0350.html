FBA (Fulfillment by Amazon) sellers depend on inventory availability. Amazon's seller-quality team measures stockout days per ASIN per month — sustained stockouts demote search ranking and signal poor inventory planning.

1. The source data — inventory_snapshots

seller_idasinsnapshot_dateqty_on_hand
S_001A_012024-01-150

2. The SQL — count zero-qty days per (seller, month)

SELECT seller_id,
  substr(snapshot_date, 1, 7) AS month,
  COUNT(DISTINCT snapshot_date) AS days_observed,
  SUM(CASE WHEN qty_on_hand = 0 THEN 1 ELSE 0 END) AS stockout_days,
  ROUND(100.0 * SUM(CASE WHEN qty_on_hand = 0 THEN 1 ELSE 0 END)
              / NULLIF(COUNT(*), 0), 1) AS stockout_rate_pct
FROM inventory_snapshots
GROUP BY seller_id, month
ORDER BY stockout_rate_pct DESC;

3. Strategic explanations

The substr trick for month bucket

SQLite has no DATE_TRUNC. substr(date, 1, 7) takes "YYYY-MM" from "YYYY-MM-DD" — works because ISO dates are length-fixed. Cross-dialect portable.

The "qty_on_hand = 0" semantic gap

Inventory tables capture different scenarios as zero:

  • Sold out — recoverable (restock)
  • Discontinued — permanent
  • Reserved (not net-available)
  • Data feed broken

Production tracks qty_on_hand AND qty_available separately. The query should use qty_available for stockout-the-customer-experiences metric.

Snapshot frequency matters

Daily snapshots = stockout-day granularity. Hourly snapshots show the % of HOURS out of stock — much higher fidelity. Match the metric grain to the snapshot grain.

The "search-rank impact" tie-back

Amazon's search-ranking model penalizes sustained stockouts. Each stockout day costs ~30% of normal sales velocity for 1-2 weeks after restock. The query result feeds the seller-warning email at > 5% stockout rate.

Seasonality adjustment

Holiday-season stockouts are common but devastating. Compare each (seller, month) to the seller's historical baseline:

... stockout_rate_pct - AVG(stockout_rate_pct) OVER (PARTITION BY seller_id) AS deviation
Senior signal: use substr for portable month-bucketing, distinguish qty_on_hand vs qty_available, match metric grain to snapshot grain, and connect stockouts to search-rank impact. Per-seller-baseline normalization is the over-the-bar slice.
