HashiCorp's supply-chain team ranks its suppliers by on-time delivery rate — the share of a supplier's shipments that arrived on schedule. The reference solution computes that rate with a CASE expression, and at a glance it looks fine. Run it against a realistic table, though, and the ranking is wrong: the CASE condition tests the wrong thing, and every supplier with no missing data scores a perfect rate even when half their shipments were late.

on_time is a boolean: true = arrived on schedule, false = late, NULL = status never recorded. Supplier 1 was on time 2 of 3 times; supplier 2 exactly half; supplier 3 perfect; supplier 4 never on time, with one NULL. A correct ranking should be 3, 1, 2, 4.

2. The SQL — on-time rate, then RANK

The reference solution:

SELECT supplier_id, AVG(CASE WHEN on_time IS NOT NULL THEN 1.0 ELSE 0 END) AS rate,
  RANK() OVER (ORDER BY AVG(CASE WHEN on_time IS NOT NULL THEN 1.0 ELSE 0 END) DESC) AS rk
FROM shipments GROUP BY supplier_id;

The bug is the condition on_time IS NOT NULL. That is true for both true and false — so a late shipment scores 1.0 just like an on-time one. The expression measures "fraction of shipments with a recorded status", not "fraction on time". The corrected version tests the boolean itself:

SELECT supplier_id,
  AVG(CASE WHEN on_time THEN 1.0 ELSE 0 END) AS rate,
  RANK() OVER (ORDER BY AVG(CASE WHEN on_time THEN 1.0 ELSE 0 END) DESC) AS rk
FROM shipments GROUP BY supplier_id;

4. Strategic explanations

The bug: IS NOT NULL instead of the boolean

CASE WHEN on_time IS NOT NULL asks "is the status known?" The metric needs "did it arrive on time?" — which is just CASE WHEN on_time THEN …, since on_time is already a boolean. With the fix, false contributes 0 (a late shipment hurts the rate) and so does NULL (CASE's WHEN NULL is not satisfied, so it falls to ELSE 0). That treats a missing status as "not confirmed on time" — a defensible, conservative default.

What to do with NULL — a real decision

Folding NULL into the denominator as a miss (above) is one choice. The other is to exclude unverified shipments and rate only confirmed ones:

SELECT supplier_id,
       AVG(CASE WHEN on_time THEN 1.0 ELSE 0 END) AS rate
FROM shipments
WHERE on_time IS NOT NULL          -- only judged shipments
GROUP BY supplier_id;

For supplier 4 that is the difference between a 0% rate (one late, one NULL counted) and a 0% rate over a single judged shipment — same number here, but very different denominators. State which definition you are using; the interviewer is listening for that.

AVG(CASE…) vs COUNT FILTER vs AVG(bool::int)

Three idioms compute the same proportion:

-- a) explicit CASE
AVG(CASE WHEN on_time THEN 1.0 ELSE 0 END)
-- b) FILTER — counts on-time, divides by all rows
COUNT(*) FILTER (WHERE on_time)::numeric / COUNT(*)
-- c) cast the boolean straight to int
AVG(on_time::int)

Watch the NULL semantics: in (a) and (c) a NULL row averages in as 0; in (b) FILTER (WHERE on_time) drops NULLs from the numerator but COUNT(*) keeps them in the denominator — same result as (a). If you wanted NULLs gone entirely you would filter in the WHERE.

The tie that RANK exposes

If two suppliers have identical rates, RANK gives them the same rank and skips the next (1, 1, 3). That is correct competition ranking. But a 100% rate from 2 shipments is not the same as 100% from 500 — rate alone is fragile on small samples. A senior answer surfaces volume as a tiebreaker or a confidence caveat:

RANK() OVER (ORDER BY on_time_rate DESC, total_shipments DESC)
Low-volume suppliers

A supplier with one lucky on-time delivery scores a perfect 1.0 and tops the board. Before ranking, consider a HAVING COUNT(*) >= n floor so the leaderboard reflects reliable suppliers, not statistical noise.

Senior signal: catch that CASE WHEN on_time IS NOT NULL measures "status recorded" not "on time" — counting late shipments as successes — and fix it to test the boolean directly; then decide deliberately whether a NULL status is a miss or an exclusion, and guard the ranking against tiny-sample suppliers with a volume floor or tiebreaker.
