TikTok Creator Fund pays based on follower-tier × engagement. The query buckets creators into nano/micro/mid/macro/mega and computes RPM (revenue per mille views).

2. The SQL — tier classification + RPM

SELECT
  CASE
    WHEN follower_count >= 1000000 THEN 'mega (1M+)'
    WHEN follower_count >= 100000  THEN 'macro (100K+)'
    WHEN follower_count >= 10000   THEN 'mid (10K+)'
    WHEN follower_count >= 1000    THEN 'micro (1K+)'
    ELSE 'nano' END AS tier,
  COUNT(*) AS creators,
  ROUND(AVG(earnings), 2) AS avg_earnings,
  ROUND(SUM(earnings), 2) AS total_earnings,
  ROUND(SUM(earnings) * 1000.0 / NULLIF(SUM(total_views), 0), 3) AS rpm
FROM creators GROUP BY tier ORDER BY avg_earnings DESC;

TikTok's RPMs are 10-20x lower than YouTube — controversial. Creators monetize via brand deals, not Fund.

Niche multiplier

Same RPM doesn't apply to all categories. Finance/business niches: 2-3x base. Lifestyle: base. Music: 0.5x. Production multiplies by category.

The anti-spam guard

Fake-view bots inflate RPM denominators. Anti-fraud filters before the rate calc.

The Creator Fund vs Creativity Program

TikTok migrated from Creator Fund (low pay) to Creativity Program (higher pay, longer-form videos required). The query above is plan-agnostic; production splits.

Senior signal: 5-tier follower bucketing with named ranges, compute RPM as the per-mille payout, cite TikTok's 10-20x lower vs YouTube reality, apply niche multipliers, and split by Creator Fund vs Creativity Program plan.
