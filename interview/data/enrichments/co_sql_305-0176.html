Cart abandonment is the single highest-leverage e-commerce metric. Reducing it by 1 percentage point on a $1B-annual store recovers $10M. Every checkout team at Amazon / Shopify / Target watches this number daily. The query is one line; the trap is what counts as "abandoned" and how to slice the metric to find the broken step.

1. The source data — sessions

8 sessions. 5 added to cart; 3 of those abandoned (didn't check out). Two browse-only sessions don't count toward the abandonment metric — they never added anything to abandon.

2. The SQL — conditional aggregation

SELECT
  100.0 * SUM(CASE WHEN added_to_cart AND NOT checked_out THEN 1 ELSE 0 END)
        / NULLIF(SUM(CASE WHEN added_to_cart THEN 1 ELSE 0 END), 0) AS abandon_rate
FROM sessions;

3. The resulting analysis

abandon_rate
60.0%

3 abandons ÷ 5 add-to-cart sessions × 100 = 60%. (Industry baseline is around 70% — this fictional store is slightly better than average.) The CRO sees this number and asks: "Where in the checkout flow are we losing them?" — which the next query answers.

4. Strategic explanations

The denominator is "added_to_cart", not all sessions

The most common bug in cart-abandonment queries: dividing by total sessions instead of carted sessions. That dilutes the metric — a store with 10x more browse-only traffic looks like its abandonment dropped, when nothing changed at the cart step. The right denominator is SUM(added_to_cart); sessions that never added anything aren't candidates for abandonment.

SQLite handles boolean columns as integers

added_to_cart AND NOT checked_out works in SQLite because BOOLEAN is stored as INTEGER (0/1). In MySQL / Postgres it works because they have native BOOL. In Snowflake also fine. The query is dialect-portable as written. Just don't expect WHERE added_to_cart = TRUE to work in older SQLite — use WHERE added_to_cart = 1 or just WHERE added_to_cart (which both engines coerce).

The funnel-step breakdown — the question that always comes next

Single-number abandonment is the headline. The actionable version slices by funnel step:

SELECT
  COUNT(*) FILTER (WHERE viewed_product)               AS step_1_viewed,
  COUNT(*) FILTER (WHERE added_to_cart)                AS step_2_added,
  COUNT(*) FILTER (WHERE entered_checkout)             AS step_3_checkout,
  COUNT(*) FILTER (WHERE entered_payment)              AS step_4_payment,
  COUNT(*) FILTER (WHERE checked_out)                  AS step_5_purchased,
  -- conversion ratios between steps
  ROUND(100.0 * COUNT(*) FILTER (WHERE added_to_cart)
        / NULLIF(COUNT(*) FILTER (WHERE viewed_product), 0), 1) AS view_to_add_pct,
  ROUND(100.0 * COUNT(*) FILTER (WHERE checked_out)
        / NULLIF(COUNT(*) FILTER (WHERE added_to_cart), 0),  1) AS add_to_buy_pct
FROM sessions;

Each step's drop-off rate localises the problem. If step_3 → step_4 drops 40%, it's a payment-flow issue; if step_2 → step_3 drops 50%, it's the cart UX or shipping reveal.

Time-segmented abandonment — for cohort experiments

When the team A/B tests a new checkout flow, slicing abandonment by experiment variant is the win condition:

SELECT variant_id,
  ROUND(100.0 * SUM(CASE WHEN added_to_cart AND NOT checked_out THEN 1 ELSE 0 END)
              / NULLIF(SUM(CASE WHEN added_to_cart THEN 1 ELSE 0 END), 0), 1) AS abandon_rate
FROM sessions JOIN ab_assignments USING (session_id)
GROUP BY variant_id;

Variant with lower abandonment wins (assuming sample size + statistical significance).

"Recovered" carts — the email-reminder layer

Many e-commerce platforms send a reminder email 1-3 hours after abandon. Recovered carts (added → abandoned → returned later → converted) flip from abandon to convert. Production query needs a last_event_ts + a 24-hour window to handle this:

-- Sessions classed as "abandoned" only if no checkout within 24h of last cart add
WHERE added_to_cart
  AND NOT EXISTS (
    SELECT 1 FROM checkouts c
    WHERE c.session_id = sessions.session_id
      AND c.checked_out_ts <= sessions.last_cart_add_ts + INTERVAL '24 hours'
  )
Senior signal: the right denominator (added-to-cart, not all-sessions) is the basic bar. Volunteering the funnel-step breakdown and the A/B variant slice is what moves the answer up. The "recovered carts" 24-hour window is over-the-bar — most candidates miss it.
