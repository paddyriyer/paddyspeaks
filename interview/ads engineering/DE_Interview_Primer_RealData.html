18. Detect & Handle Data Skew L5

Problem: A join is slow because one key has a disproportionate share of rows. Detect and fix.

SQL-- Step 1: Detect skew
SELECT join_key, COUNT(*) AS rows,
  ROUND(100.0 * COUNT(*) / SUM(COUNT(*)) OVER(), 2) AS pct_of_total
FROM skewed_table
GROUP BY join_key
ORDER BY rows DESC;

1. COUNT(*) & COUNT(col)
SQL
SELECT COUNT(*) AS all_rows,
       COUNT(amount) AS non_null
FROM orders;
Python — Manual loop
all_rows = 0
non_null = 0
for o in orders:
    all_rows += 1
    if o["amount"] is not None:
        non_null += 1
Python — Idiomatic
all_rows = len(orders)
non_null = sum(1 for o in orders
               if o["amount"] is not None)
Result on orders.csv (500 rows): all_rows = 500, non_null = 500
Gotcha: COUNT(*) counts rows including NULLs. COUNT(col) skips NULLs. In Python, len() counts everything; to mimic COUNT(col), filter explicitly.
2. SUM
SQL
SELECT SUM(amount) FROM orders;
Python — Manual loop
total = 0.0
for o in orders:
    total += o["amount"]
Python — Idiomatic
total = sum(o["amount"] for o in orders)
Result on orders.csv (500 rows): $46,004.18
SQL trap: SUM of an empty set is NULL, not 0. Python's sum() returns 0. Wrap with COALESCE(SUM(x), 0) if you need 0.
3. AVG
SQL
SELECT AVG(amount) FROM orders;
Python — Manual loop
total = 0.0
n = 0
for o in orders:
    total += o["amount"]
    n += 1
avg = total / n
Python — Idiomatic
from statistics import mean
avg = mean(o["amount"] for o in orders)
# or: sum(amts) / len(amts)

NULL handling: SQL AVG ignores NULLs (SUM(x)/COUNT(x), not SUM(x)/COUNT(*)). Python statistics.mean will error on an empty input — guard with a check.

2. Building a Data Quality Pipeline (Airflow-style) L4–L5

Problem: Wire up a production-grade data quality system step by step. Start with one assertion, build out a check suite, add severity-based alerting, orchestrate as a DAG, and finally add remediation so the pipeline survives partial failures.

Why this lives in the L5 portion of the interview: Anyone can write assert len(df) > 0. The question is what happens at 3 AM when a single check fails — does the pipeline halt? Does oncall get paged for every NULL email? Does the bad row poison the whole batch? This section walks through the five-step build that handles all of that, with every check actually running against the practice CSVs.
Step 1 One assertion — does the table look right at all?

Step 2 A reusable suite — six check primitives, applied broadly

The shape that scales: each check primitive returns (ok, detail); a runner wraps it for safe execution; the actual suite is just a list of compositions. Now the same pattern handles volume, freshness, null/uniqueness, referential integrity, and business rules.

PYTHON# Step 2: Composable check functions + a runner
from dataclasses import dataclass

@dataclass
class CheckResult:
    name: str
    status: str        # PASS | FAIL | ERROR
    detail: str

def run_check(name, check_fn):
    try:
        ok, detail = check_fn()
        return CheckResult(name, 'PASS' if ok else 'FAIL', detail)
    except Exception as e:
        return CheckResult(name, 'ERROR', f'{type(e).__name__}: {e}')

# 6 reusable check primitives covering the 'standard' DQ surface area
def check_row_count(rows, min_rows):
    n = len(rows); return n >= min_rows, f'{n:,} rows (min: {min_rows:,})'

def check_freshness(rows, date_field, max_age_days, ref_date):
    latest = max(r[date_field] for r in rows)
    age = (ref_date - latest).days
    return age <= max_age_days, f'latest: {latest} ({age} days old)'

def check_no_nulls(rows, field):
    nulls = sum(1 for r in rows if not r.get(field))
    return nulls == 0, f'{nulls} null values in {field}'

def check_unique(rows, field):
    seen = Counter(r[field] for r in rows)
    dupes = [k for k, v in seen.items() if v > 1]
    return len(dupes) == 0, f'{len(dupes)} duplicate {field} values'

def check_referential(child, fk, parent, pk):
    parent_keys = {r[pk] for r in parent}
    orphans = [r for r in child if r[fk] not in parent_keys]
    return len(orphans) == 0, f'{len(orphans)} orphan rows'

def check_business_rule(rows, name, predicate):
    fails = [r for r in rows if not predicate(r)]
    return len(fails) == 0, f'{len(fails)} rows violate {name}'

Step 4 Orchestrate as a DAG — let independent branches run in parallel

A linear list of checks wastes time and creates false correlations: a freshness failure has no reason to block schema validation. Structure the checks as a DAG so independent branches run in parallel; a critical failure only skips its downstream stages, not the whole graph.

5. Topological Sort — Pipeline DAG L4

Problem: Schedule pipeline stages so every stage runs after its dependencies.

Pythonfrom collections import defaultdict, deque

def topo_sort(deps):  # deps = {node: [parents]}
    children = defaultdict(list)
    in_deg = {n: 0 for n in deps}
    for n, parents in deps.items():
        for p in parents:
            children[p].append(n); in_deg[n] += 1
    ready = deque(n for n, d in in_deg.items() if d == 0)
    order = []
    while ready:
        n = ready.popleft(); order.append(n)
        for ch in children[n]:
            in_deg[ch] -= 1
            if in_deg[ch] == 0: ready.append(ch)
    return order

Pythonimport functools, time

def retry(max_tries=3, delay=1, exceptions=(Exception,)):
    def decorator(func):
        @functools.wraps(func)
        def wrapper(*args, **kwargs):

13. Rate Limiter (Token Bucket) L4

Problem: Throttle calls at a steady rate with burst capacity.

Pythonimport time

class TokenBucket:
    def __init__(self, capacity, refill_per_sec):
        self.capacity = capacity
        self.tokens = capacity
        self.rate = refill_per_sec
        self.last = time.monotonic()
    def take(self, n=1):
        now = time.monotonic()
        self.tokens = min(self.capacity, self.tokens + (now - self.last) * self.rate)
        self.last = now
        if self.tokens >= n:
            self.tokens -= n; return True
        return False

20. Deque Sliding Window Max L4

Problem: Find the max of every contiguous k-window in O(n).

Pythonfrom collections import deque

def sliding_max(arr, k):
    dq, out = deque(), []
    for i, v in enumerate(arr):
        while dq and arr[dq[-1]] <= v: dq.pop()
        dq.append(i)
        if dq[0] <= i - k: dq.popleft()
        if i >= k - 1: out.append(arr[dq[0]])
    return out
