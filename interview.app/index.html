    FAQ

    How many interview questions does the bank include?
    The bank ships 1511 hand-curated interview questions across SQL, Python, Snowflake and Git, sourced from real interview rounds at 105 companies including Google, Amazon, Meta, Apple, Atlassian and Anthropic.
    Do I need an account to use the playgrounds?
    No. Both the SQL and Python playgrounds run entirely in your browser. There is no sign-up, no backend, and no telemetry. Your code stays on your device.
    How do the playgrounds work without a server?
    The SQL playground uses sql.js (SQLite compiled to WebAssembly) with synthetic per-question schemas auto-generated from the inferred schema. The Python playground uses Pyodide (CPython 3.12 in WebAssembly) with the standard library pre-imported and pandas/numpy available on demand.
    Can I save the questions I want to practise?
    Yes. Tick the boxes on any question card to add it to your set. Your selections persist in localStorage and can be exported as JSON, Markdown or a printable PDF.
    Are the SQL solutions guaranteed to run in SQLite?
    Most of the 985 SQL solutions run as-is in the in-browser SQLite. Those that use Snowflake- or PostgreSQL-only features (QUALIFY, INTERVAL '7 days', regex functions, …) show a banner with the specific incompatible token so you can read the reference and run it in your target database.
    How do I report a bad question or solution?
    Open an issue at github.com/paddyriyer/paddyspeaks. The dataset and every override are stored as plain JSON files under interview/data/, so contributions are trivial.
    How is this different from LeetCode, HackerRank, StrataScratch or DataLemur?
    The bank is data-engineer-first: SQL, Python, Snowflake, Git, streaming and SCDs — not the algorithmic-puzzle catalog LeetCode and HackerRank optimize for. Unlike StrataScratch and DataLemur it is fully free, runs entirely client-side (no account, no telemetry), and ships a 23-deep-dive data-modeling track (ride-share, ads, payments, streaming, marketplaces, SaaS) you don't get on a question-only platform.
    Does this cover senior, staff and system-design data engineer interviews?
    Yes — the Design track is built for senior and staff data engineer interview rounds. It covers Snowflake architecture (micro-partitions, clustering, time travel), streaming systems (Kafka, Flink, exactly-once semantics), data quality and reliability patterns, M&A data integration, hot-shard and skew mitigation, and ML-engineering data interview prep — each as a deep-dive with stakeholder-grouped SQL and worked examples.
    What SQL topics does the bank cover?
    Window functions, CTEs and recursive CTEs, growth accounting and retention cohorts, SCD Type 2 modeling, percentile and median computations, sessionization with gaps-and-islands, top-N per group, funnel analysis, time-series rollups, dedup with QUALIFY, and Snowflake-specific patterns. Each question ships an inferred schema, reference solution, and runs in the in-browser SQLite playground.

    1511 questions sourced from 105 companies including Google, Amazon, Meta, Apple, Microsoft, Netflix, Atlassian, Anthropic, DoorDash, Uber, Airbnb, Stripe, Snowflake, Databricks and Snap. Filter the bank by company to drill into a specific employer's interview style and difficulty curve.
