# How to use this dashboard to make decisions

The redesign is organized so each section answers one question and points to one
action. Read it top-down; stop when you have your decision.

## The six questions → where to look
| Question | Tab | What to read |
|---|---|---|
| What changed? | **Overview** | KPI cards vs the previous equal period (Δ% + absolute) |
| Why did it probably change? | **Overview → "What deserves attention?"** | Rule-based insights (observation → inference → action → confidence) |
| Which sources brought value? | **Acquisition** | Sort by **engagement rate / conversion**, not visits. Watch the value classes. |
| Where do visitors struggle? | **Content** & **Interview Studio** | Exit rate, scroll drop-off, funnel abandonment point |
| What's working? | **Content** (2×2) & **Interview Studio** (per-track completion) | Winners / hidden gems; tracks with repeat use |
| What next? | Every tab's action line + **Journeys/Retention** | Promote, improve, or build |

## Decisions this dashboard is built to support
- **What content to promote** → Content tab, **Hidden gems** (low reach, high
  engagement). These are your best ROI on a LinkedIn post.
- **What content to improve** → **Click magnets** (high reach, weak
  engagement) and pages with steep 50%→90% scroll drop-off.
- **Which sources bring serious readers** → Acquisition, **low-volume/
  high-value** sources. Do more of what those channels did.
- **Where visitors abandon Interview Studio** → Studio funnels; the
  "abandonment point" is the first step with the biggest drop. Fix that screen.
- **Which tracks create repeat use** → Studio → repeat usage + Retention
  cohorts by track. Invest content there.
- **What to build next** → Studio → **Learning Opportunities**: topics users get
  wrong, and **search terms with no results** (`no_search_results`) = demand you
  haven't met yet.

## Reading the numbers honestly
- **Is a change real or noise?** Every rate shows its **sample size**. If the
  denominator is below the small-sample floor (default 30) it's flagged and the
  insight engine will not draw a conclusion from it. Treat flagged moves as
  provisional.
- **Medians, not averages.** Engaged time and scroll headline as medians so a
  few very long sessions don't mislead you. Mean is shown alongside for context.
- **Engaged single-page reads are not bounces.** A long article read to 90% is a
  success, not a bounce — the engagement rate reflects that.
- **Empty retention cells show "—", not 0.** A Day-30 cell is blank until 30
  days have actually elapsed for that cohort. Don't read a blank as failure.
- **Bots are visible, not hidden.** Suspected automated traffic is flagged and
  excluded from headline metrics but still inspectable in Data Quality, so you
  can trust the human numbers and audit the rest.

## A weekly 5-minute routine
1. Overview: skim the KPI deltas and the top insight. One thing changed — what?
2. Acquisition: did a source's **quality** (not volume) shift?
3. Content: any new hidden gem to promote, or click magnet to fix?
4. Studio: did the funnel abandonment point move? Any new no-result searches?
5. Pick **one** action from the highest-priority insight. Ship it. Re-check next week.
